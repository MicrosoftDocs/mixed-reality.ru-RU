# <a name="testing-your-app-on-hololens"></a>Тестирование приложения на HoloLens

Тестирование приложений HoloLens очень похожи на тестирование приложений Windows. Все обычные области можно рассматривать (функции, возможности взаимодействия, производительности, безопасности, надежности, и т.д.). Тем не менее, существуют некоторые области, требующие специальной обработки или внимание к деталям, которые обычно не наблюдается в приложениях для компьютера или телефона. Holographic приложения должны работать без проблем в разнообразных средах. Также необходима для обеспечения производительности и улучшение комфорта все время. Руководства по проверке этих областей, описанные в этом разделе.

## <a name="performance"></a>Производительность

Holographic приложения должны работать без проблем в разнообразных средах. Также необходима для обеспечения производительности и улучшение комфорта все время. Поэтому для пользовательского интерфейса с помощью Holographic приложения, что у нас есть весь раздел посвящен его важна производительность. Убедитесь в том, прочтите и выполните [основные сведения о производительности для смешанной реальности](understanding-performance-for-mixed-reality.md)

## <a name="testing-3d-in-3d"></a>Тестирование 3D в трехмерном пространстве
1. **Протестируйте приложение в различных пробелами максимально.** Попробуйте в больших комнаты, небольшой комнаты, ванные комнаты, машины, спальни, офисов и т. д. Также учитывает комнаты рассмотрения с помощью таких не вертикальные стены, изогнутой стены, максимальных под нестандартных функций. Она работает также в том случае, при переходе между комнаты, пол, через коридорах зданий или лестницы?
2. **Протестируйте приложение в различным условиям.** Она отвечает должным образом для различных условий окружающей среды, такие как освещения, черный поверхности, прозрачного/Светящийся поверхности, например зеркала, стены стекла и т. д.
3. **Тестирование приложения в условиях перемещения.** Вставьте на устройстве и повторите ваших сценариев в различных состояниях движения. Она отвечает должным образом для различных перемещения или устойчивого состояния?
4. **Проверьте работу приложения под различными углами зрения.** При наличии голограмма world блокировки, что произойдет, если ваш пользователь выходит за ней? Что произойдет, если что-то идет между пользователем и голограмма? Что делать, если пользователь смотрит голограмма выше или ниже?
5. **Используйте Пространственные и звуковые подсказки.** Убедитесь, что ваше приложение использует их для предотвращения потери пользователя.
6. **Тестирование приложения на разных уровнях шума.** Если вы реализовали голосовые команды, попробуйте вызывает их с различными уровнями шума.
7. **Протестируйте свои приложения сидели и положение**. Убедитесь в том, что тестирование из мест и положение позиций.
8. **Тестирование приложения на различных расстояниях**. Элементы пользовательского интерфейса для чтения и взаимодействовал издалека? Выполняет react вашего приложения для пользователей, получающих слишком близко к вашей голограммы?
9. **Тестирование приложения от распространенных взаимодействия приложений строке**. Все плитки приложений и 2D универсальных приложений имеют [панели приложения](navigating-the-windows-mixed-reality-home.md#moving-and-adjusting-apps) , позволяет контролировать, как приложение размещается в смешанной среде. Убедитесь, что нажатием корректно завершает процесс приложения, и что кнопку "Назад" поддерживается в контексте 2D универсального приложения. Попробуйте масштабирование и перемещение приложения в [режим Adjust](navigating-the-windows-mixed-reality-home.md#moving-and-adjusting-apps) при активном и хотя плитки приостановленного приложения.

## <a name="comfort"></a>Комфорта
1. **Обрезать плоскости.** Удалось Внимательный куда [голограммы подготавливаются к просмотру](hologram-stability.md#hologram-render-distances).
2. **Избегайте виртуального перемещения согласована фактическое перемещения.** Избегайте перемещения камеры способом, который не является типичным для фактического перемещения пользователя. Если вашему приложению требуется, перемещении пользователей через сцены, сделать движение предсказуемым, свести к минимуму ускорение и позволить пользователю управлять перемещение.
3. **Следуйте рекомендациям голограмма качества.** Высокопроизводительных приложений, которые реализуют [голограмма качества рекомендации](hologram-stability.md) вероятность, с последующим получением discomfort пользователя.
4. **Распространение голограммы по горизонтали, а не по вертикали.** Принудительная пользователя проводить продолжительного периода времени, поиск, или вниз может привести к усталости в шее.

## <a name="input"></a>Ввод

### <a name="gaze-and-gestures"></a>Взглядом и жестов

[Помощи](gaze.md) является базовой ввода на HoloLens, позволяя пользователю призваны голограммы и среды. Визуально отобразится которой предназначен ваш взглядом на основе положения курсора. Довольно часто, чтобы связать курсор взглядом с курсором мыши.

[Жесты](gestures.md) , как взаимодействовать с голограммы, таких как щелчок мыши. В большинстве случаев поведение мыши и сенсорного ввода одинаковы, но очень важно для понимания и проверки, если они отличаются.

**Проверьте, если приложение имеет другое поведение с помощью мыши и сенсорного ввода.** Это будет выявление несоответствий, которые проектные решения, чтобы сделать работу более естественным для пользователей. Например активируя действие на основании при наведении курсора мыши.

> [!NOTE]
> Дополнительные рекомендации, относящиеся к HoloLens 2 [ожидается в ближайшее время](index.md#news-and-notes).

### <a name="custom-voice-commands"></a>Пользовательские голосовые команды

[Голоса](voice-input.md) является естественной форме взаимодействия. Взаимодействие с пользователем может быть магического или сбивающим с толку в зависимости от выбора команды и как они будут предоставлены. Как правило не следует использовать голосовые команды системы, например «Select» или «Привет, Кортана» как пользовательские команды. Вот несколько моментов, которые следует учитывать.
1. **Избегайте использования команд, которые звучат одинаково.** Это потенциально может активировать Неверная команда.
2. **Выберите фонетически форматированного слов, когда это возможно.** Это будет свести к минимуму и избежать false активаций.

### <a name="peripherals"></a>Периферийные устройства

Пользователи могут взаимодействовать с приложения с помощью [периферийные устройства](hardware-accessories.md). Приложения не нужно делать ничего особенного, чтобы воспользоваться преимуществами этой возможности, тем не менее есть несколько вещей, стоит посмотреть.
1. **Проверка пользовательского взаимодействия.** Такие вещи, как пользовательских сочетаний клавиш для вашего приложения.
2. **Проверка переключения типов входных данных.** Попытка использовать несколько методов ввода для выполнения задачи, такие как voice, жест, мыши и клавиатуры в тот же сценарий.

## <a name="system-integration"></a>Интеграция системы

### <a name="battery"></a>Аккумулятор

Протестируйте приложение без источника питания, подключенных к понять, насколько быстро он выполняет сток аккумулятора. Один могут легко понять состояние батареи, просмотрев показания Индикатор питания. ![Состояния Индикаторов, которые указывают заряд батареи](images/batterypowerledindication-500px.png)

### <a name="power-state-transitions"></a>Переходы состояния питания

Проверьте ключевых сценариев работы должным образом, когда переходов между состояниями питания. Например приложения остаются в исходном положении? Она правильно хранит свое состояние? Она продолжать функционировать должным образом?
1. **Ждущий / Resume.** В режим ожидания, один клавишу и сразу же нажмите кнопку питания. Устройство также перейдет в ждущий режим автоматически после 3 минут бездействия. Чтобы возобновить работу из ждущего режима, один нажмите и сразу же нажмите кнопку питания. Устройство также будет возобновлена при подключении или отключить его от источника питания.
2. **Завершение работы или перезапуска.** Для завершения работы удерживая кнопку питания постоянно 6 секунд. Чтобы перезапустить, нажмите кнопку питания.

### <a name="multi-app-scenarios"></a>Сценарии с несколькими приложениями

Проверьте основные функциональные возможности приложения, при переключении между приложениями, особенно в том случае, если вы реализовали в фоновом режиме. Интеграция Кортаны и вставьте также являются стоит посмотреть, где это возможно.

## <a name="telemetry"></a>Телеметрия

Использование телеметрии и аналитики по использованию. Интеграция аналитики в приложение поможет вам получить ценные сведения о приложении тестерам бета-версий и конечных пользователей. Эти данные можно использовать для оптимизации приложения, а затем отправления Store, а также для последующих обновлений. Предусмотрено много вариантов analytics там. Если вы не уверены, с которого следует начать, ознакомьтесь с [App Insights](https://www.visualstudio.com/products/application-insights-vs.aspx).

Рассмотрите следующие вопросы:
1. Как пользователи используют пространство?
2. Как такое приложение размещения объектов на свете — обнаружить проблемы?
3. Сколько времени они затрачивают на разных этапах приложения?
4. Сколько времени они затрачивают на приложение?
5. Каковы наиболее распространенные пути использования пользователи пытаются?
6. Пользователи столкнулись непредвиденное состояние и ошибки?

## <a name="emulator-and-simulated-input"></a>Эмулятор и имитации ввода

[HoloLens эмулятор](using-the-hololens-emulator.md) — отличный способ эффективного тестирования Holographic приложения с различными характеристиками корректировки моделируемой пользовательской и пробелы. Ниже приведены советы по эффективной с помощью эмулятора для тестирования приложения.
1. **Чтобы развернуть тестирования, воспользуйтесь виртуальные комнаты эмулятора.** Эмулятор поставляется с набором виртуального комнаты, которые можно использовать для тестирования приложений в средах, еще больше.
2. **Использование эмулятора, рассмотрим приложение со всех сторон.** Ключи PageUp/PageDn сделает Ваш корректировки моделируемой пользовательской высоты.
3. **Тестирование приложения с реальной HoloLens.** Эмулятор HoloLens — отличное средство для быстрого перебирайте приложения и перехвата новых ошибок, но убедитесь, что вы также протестировать на физических HoloLens перед отправкой в Windows Store. Это важно для обеспечения отлично выглядит на реальном оборудовании и эффективности работы.

## <a name="automated-testing-with-perception-simulation"></a>Автоматическое тестирование с помощью моделирования восприятие

Некоторые разработчики приложений может потребоваться автоматизации тестирования пользовательских приложений. Помимо простых модульных тестов, можно использовать [perception моделирования](perception-simulation.md) стека в HoloLens для автоматизации человеком, так и все входные данные для приложения. API моделирования восприятия можно отправить имитации входных данных для HoloLens эмулятора или физического HoloLens.

Если приложение предназначено для семейства устройств Windows.Holographic, [комплект сертификации приложений для Windows](https://msdn.microsoft.com/library/windows/apps/xaml/mt186449.aspx) локального статического анализа тестов будет выполняться только на вашем Компьютере. Тесты не будут запущены на вашей HoloLens.
