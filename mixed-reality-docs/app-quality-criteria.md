# <a name="app-quality-criteria"></a>Критерии контроля качества приложения

В этом документе описываются top факторы, влияющие на качество приложения смешанной реальности. Для каждого фактора предоставляется следующая информация
* Обзор — краткое описание коэффициент качества и почему она важна.

* Критерии качества, как вычислить коэффициент качества.
* Как измерить — методы для измерения (или возникнуть) проблема.
* Рекомендации — Сводка подходов для повышения удобства работы пользователя.
* Ресурсы — соответствующие разработчика и ресурсы для разработки, которые полезны для создания лучших результатов приложения.

## <a name="frame-rate"></a>Частота кадров

Частота кадров — это главным условием голограмма комфорта стабильность и пользователя. Частота кадров ниже рекомендуемые целевых объектов может привести к голограммы отображаются перебои управления, отрицательно влияет на believability работы и что может привести к усталости глаз. Частота кадров целевого для работы в Windows Mixed Reality иммерсивную будет либо или 60 Гц 90, в зависимости от того, какие смешанной реальности совместимых компьютеров под управлением Windows, которая должна поддерживаться. Для HoloLens частота кадров целевого — 60 Гц.

### <a name="how-to-measure"></a>Как измерить

* График частота кадров в режиме реального времени предоставляется через [Windows Device Portal](using-the-windows-device-portal.md#system-performance) в разделе «Производительность системы».
* Для отладки разработки, добавьте счетчик диагностики частота кадров в приложение. Ознакомьтесь с ресурсами образца счетчика.
* Падения частоты кадров может возникать в устройстве во время работы приложения, перемещая голове в сторону. Если она отображается непредвиденный перебои управления перемещения, затем низкая частота кадров или стабильности плоскости она может вызвать.

### <a name="recommendations"></a>Рекомендации

* Добавьте счетчик частота кадров в начале процесса разработки.
* Изменения, которые повлечь Снижение частоты кадров следует вычислить и соответствующим образом равна устранения ошибки.

## <a name="hologram-stability"></a>Голограмма стабильность

Стабильные голограммы повысить удобство использования и believability приложения и создавать более комфортную работу Просмотр для пользователя. Качество голограмма стабильности является результатом хорошее приложение разработки, а также возможность устройства понять (отслеживание) его среды. Частота кадров является главным условием стабильность, другие факторы могут повлиять на стабильность включая:

* Использование плоскости стабилизации
* Расстояние до пространственных привязки
* Отслеживание

### <a name="how-to-measure"></a>Как измерить

Хотя наши устройства и условия для просмотра:

* Переместить головой из стороны в сторону, если голограммы двигаться Непредвиденная затем низкая частота кадров или вероятной причиной является неправильное выравнивание стабильности плоскости на плоскости.
* Перемещение по голограммы и среды, обратите внимание на поведение, например различных и jumpiness. Этот тип движения, скорее всего связано с устройства, не отслеживая среды или расстояние для привязки пространственных.
* При большом или нескольких голограммы во фрейме, наблюдать за поведением голограмма в различные глубины, хотя перемещение на вашей головной позицию в сторону, если shakiness отображается, это может вызвано стабилизации плоскости.

### <a name="recomendations"></a>Recomendations

* Добавьте счетчик частота кадров в начале процесса разработки.
* С помощью стабилизации плоскости.
* Всегда выводить привязанного голограммы в трех ваттметров их привязки.
* Убедитесь, что ваша среда настроена для правильного отслеживания.
* Проектирование работы во избежание голограммы на различных уровнях фокальной глубины внутри фрейма.

## <a name="use-of-spatial-sound"></a>Использование пространственных звука

В Windows Mixed Reality ядро аудио предоставляет звукового компонента работы смешанной реальности, имитируя 3D звука с помощью направление, расстояние и моделирование окружающей среды. Использование пространственных звука в приложении позволяет разработчикам convincingly поместите звуков в 3 мерном пространстве (сфера) по всему пользователя. Затем эти звуки покажется так, как если бы они поступали от физического объектов или голограммы смешанной реальности в окружения пользователя. Пространственные звук — это мощное средство для общения, доступности и проектирование взаимодействия с Пользователем в приложениях смешанной реальности.

## <a name="room-scanning"></a>Сканирование комнаты

Приложения, которым требуются пространственное сопоставление данных полагаться на устройстве, чтобы автоматически собирать эти данные со временем и во всех сеансах, как пользователь изучает их среды с устройством active. Полноты и качества этих данных зависит от ряда факторов, включая количество исследований, которые пользователь выполнит, сколько времени прошло с момента просмотра и ли объекты, такие как мебель и двери были перемещены, так как устройство сканирования области. Во многих приложениях проанализирует пространственное сопоставление данных в начале работы, имела ли пользователь должен выполнить дополнительные действия для повышения качества пространственных карты и полноты. Если пользователь является необходимым условием для поиска в среду, очевидно, что рекомендации, которые необходимо указать во время сканирования.

Индикаторы направления предупреждения пользователя важное содержимое и советы для содержимого относительно его положения. Рекомендации, которые не видимого содержимого может принимать форму звуковой отправители, направленными стрелками или прямой визуальные подсказки.
