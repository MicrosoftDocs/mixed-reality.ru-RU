# <a name="add-custom-home-environments"></a>Добавление пользовательских сред home

>[!NOTE]
>Это экспериментальная функция. Испытайте их готовность к работе с ним, но не удивляйтесь, если все, что совсем не работает должным образом. Мы оценке жизнеспособности этой функции и интерес к использования, поэтому сообщите нам о вашей работы (и все ошибки, вы нашли) в [форумы для разработчиков](https://forums.hololens.com/categories/custom-home-environments).

Начиная с [обновления Windows 10 апреля 2018 г.](#release-notes-april-2018.md), мы реализовали это экспериментальная функция, которая служит для добавления пользовательских сред для выбора места (в меню "Пуск") для использования в качестве [Windows Mixed Reality домашней](#navigating-the-windows-mixed-reality-home.md). Windows Mixed Reality имеет две среды по умолчанию, Александр дома и Skyloft, которые можно выбрать в качестве домашней сети. Создание пользовательских сред позволяет расширить этот список с собственные операции создания. Мы предоставляем это в состоянии ранней оценкой интересами, от авторов и разработчиков, узнайте, какие виды миров создать и основные сведения о работе с помощью различных средств разработки.

При использовании настраиваемой среды, вы заметите, что teleporting, взаимодействие с приложениями и помещения голограммы работает так же, как в дом Клифф и Skyloft. Поиск в Интернете в сфере воображаемых также можно заполнить голограммы будущего города, — возможности безграничны!

## <a name="trying-a-sample-environment"></a>Попытки пример среды

Мы создали пример среды, демонстрирует некоторые возможности для творчества пользовательских сред home. Выполните следующие действия, чтобы опробовать его.
1. [Загрузить наш пример среды остров воображаемых](https://download.microsoft.com/download/B/2/5/B25C1AEF-40CD-4B03-A596-4BCA3D33035A/Fantasy_Island.exe) (указывает ссылка в самораскрывающийся исполняемый файл).

    ![Остров воображаемых пример среды](images/FantasyLand.jpg)<br>
    *Остров воображаемых пример среды*<br>

2. Запустите **Fantasy_Island.exe** скачанный файл.

    > [!NOTE]
    > При попытке запустить файл .exe, загруженные из Интернета (например, такой), может появиться всплывающее окно «Windows защищенного компьютера». Чтобы выполнить Fantasy_Island.exe это всплывающее окно, выберите **Дополнительные сведения о** и затем **выполнить в любом случае**. Этот параметр безопасности предназначен для защиты от загрузки файлов, может потребоваться доверия, поэтому только выберите этот параметр, если вы доверяете источнику файла.

3. Откройте **проводнике** и перейдите в папку «среды», вставьте следующий код в адресной строке: `%LOCALAPPDATA%\Packages\EnvironmentsApp_cw5n1h2txyewy\LocalState`.
4. Скопируйте пример среды, загруженный в эту папку.
5. Перезапустите **смешанный портала на самом деле**. Это обновит список сред в средстве выбора мест.
6. Включить ваш гарнитуры. Когда вы будете в домашних, откройте **меню "Пуск"** с помощью Windows кнопку вашего контроллера.
7. Выберите **местах** значок над списком закрепленных приложений, чтобы выбрать в домашней среде.
8. Вы найдете остров воображаемых среды, загруженный в списке адресов. Выберите **остров воображаемых** для ввода новой настраиваемой домашней среды!

## <a name="creating-your-own-custom-environment"></a>Создание полной среды

Помимо использования наших примеров окружений, вы можете экспортировать пользовательских сред с помощью любимых трехмерной редактирования программного обеспечения. 

### <a name="modeling-guidelines"></a>Рекомендации по моделированию

При моделировании среды, соблюдайте следующие рекомендации. Это поможет обеспечить порождает пользователя в правильной ориентации в мире believably размера:

1. Пользователи появится в центре 0,0,0 так ваше расположение требуемой spawn вокруг начала координат.
2. Работа единицы должно быть присвоено метров, таким образом, чтобы ресурсы могут быть созданы в масштабе по всему миру.
3. Доступ к оси должно быть присвоено значение «Y».
4. Ресурс должен сталкиваются «forward» к положительным оси Z.
5. Все сетки не обязательно должны быть объединены, но рекомендуется, если вы ориентируетесь на устройствах с ограниченными ресурсами.

### <a name="exporting-your-environment"></a>Экспорт среды

Windows Mixed Reality зависит от двоичных glTF (.glb), как формат доставки активов для сред. glTF являются платными свободного открытый стандарт для доставки трехмерных активов, обслуживается Khronos группы. По мере развития glTF как отраслевой стандарт для 3D-содержимого с возможностью взаимодействия в приложениях Windows и интерфейсы дадут корпорации Майкрософт поддержка формата.

Первым шагом при экспорте активы для использования в качестве пользовательских сред домашней создает модель glTF 2.0. Рабочая группа glTF поддерживает [список поддерживаемых экспорта и преобразователями](https://github.com/KhronosGroup/glTF/blob/master/README.md#converters-and-exporters) для создания модели glTF 2.0. Чтобы приступить к работе, используйте один из программ, перечисленных на этой странице для создания и экспортировать модели glTF 2.0, или преобразовать существующей модели с помощью одного из поддерживаемых конвертеры.

Кроме того, прочтите [в этой статье полезные](https://www.khronos.org/blog/art-pipeline-for-gltf) которого представлен обзор рабочего процесса art для экспорта модели glTF из Blender и 3DS Max напрямую. 

### <a name="environment-limits"></a>Ограничения среды

Все среды должно быть < 256 МБ. Для загрузки и переключиться на пустой мира с только что skybox по умолчанию, окружающие пользователя не удастся сред, размер которых превышает 256 МБ. Следует помните этот предельный размер файла при создании модели. Кроме того Если вы планируете оптимизация среды с помощью WindowsMRAssetConverter, как описано ниже, помните, что размер текстуры будут расти по мере оптимизатор запросов создает текстуры, которые имеют размер файла, но загружаются быстрее. 

### <a name="optimizing-your-environment"></a>Оптимизация среды.

Смешанная реальность Windows поддерживает ряд необязательно оптимизации, которые также значительно сократить время загрузки среды. Это может быть особенно важно для сред с помощью многих текстур, как они иногда истекло время ожидания при загрузке. Как правило, мы рекомендуем этот шаг для всех средств, тем не менее, в небольших средах с несколькими или с низким разрешением текстуры не всегда требовать его. 

Чтобы упростить этот процесс, мы создали [Windows смешанной реальности активов преобразователя (доступно на сайте GitHub)](https://github.com/Microsoft/glTF-Toolkit/releases) для проведения вашей оптимизации. Это средство использует набор служебных программ, доступных в наборе средств Microsoft glTF для оптимизации любой стандартный 2.0 glTF или .glb за счет упаковки дополнительных текстур, сжатия и уменьшении размера разрешения. 

Преобразователь в настоящее время поддерживает ряд флагов для настройки такого же поведения оптимизации. Рекомендуется выполнить, используя следующие флаги для получения наилучших результатов:

Flag|Рекомендуемые значения|Описание
---|---|---
-max-texture-size|1024 или 2048| Настроить это для повышения качества текстур, значение по умолчанию — 512 x 512. Обратите внимание, что более существенно повлияет на размер файла среды поэтому помните 256 МБ
версия - min|1803|Пользовательские среды поддерживаются только в версиях windows > = 1803. Этот флаг удалит текстуры для более старых версий и уменьшить размер файла окончательной актива

Пример:

```cmd
WindowsMRAssetConverter FileToConvert.gltf -max-texture-size 1024 -min-version 1803
```

## <a name="troubleshooting-and-tips"></a>Устранение неполадок и советы

### <a name="how-do-i-change-the-name-of-the-environment"></a>Как изменить имя среды?

Имя файла в папку «среды» будет использоваться в средстве выбора мест. Для изменения имени среды просто переименование среды имя файла и затем перезапустите смешанной реальности портал.

### <a name="how-do-i-remove-custom-environments-from-my-places-picker"></a>Как удалить пользовательских сред из моей палитры местах?

Чтобы удалить пользовательскую среду, откройте папку «среды» на Компьютере (`%LOCALAPPDATA%\Packages\EnvironmentsApp_cw5n1h2txyewy\LocalState`) и удалите среду. После перезапуска смешанной реальности портал, эта среда больше не будет отображаться в средстве выбора мест. 

### <a name="how-do-i-default-to-my-favorite-custom-environment"></a>Как default моих любимых настраиваемой среды?

Сейчас нельзя изменить среду по умолчанию. Каждый раз при запуске портала смешанной реальности, вы вернетесь в среде со скалы дома. 

### <a name="i-spawn-into-a-blank-space"></a>Я породить в пустое пространство

Windows Mixed Reality [не поддерживает среды, в которых превышает 256 МБ](#environment-limits). Когда среде превышает это ограничение, вы перейдете на пустой sky комплект без модели.

### <a name="it-takes-a-long-time-to-load-my-environment"></a>Он занимает много времени для загрузки моей среды

Можно добавить необязательный оптимизации в среде, чтобы сделать его быстрее. См. в разделе [«Оптимизация среды»](#optimizing-your-environment) подробные сведения.

### <a name="the-scale-of-my-environment-is-incorrect"></a>Неверный масштаб моей среды

Windows Mixed Reality преобразует glTF единиц метра при загрузке среды. Если среды загружает непредвиденные масштабирования, проверьте соответствие всем требованиям вашей программы экспорта, чтобы убедиться, что вы моделируете в масштабе 1 метра. 

### <a name="the-spawn-location-in-my-environment-is-incorrect"></a>Неверное расположение spawn в моей среде

Spawn расположение по умолчанию расположен в 0,0,0 в среде. Его не в настоящее время можно настроить это расположение, необходимо изменить точку spawn путем экспорта среды к серверу-источнику, расположенный в точке, требуемой spawn.

### <a name="the-audio-doesnt-sound-correct-in-the-environment"></a>Звук звучит правильный в среде

При создании пользовательской среды, он будет использовать моделирование отрисовки шума, которое не соответствует физического пространства, которую вы создали. Звук, могут поступать из так направлениях и может показаться muffled.
