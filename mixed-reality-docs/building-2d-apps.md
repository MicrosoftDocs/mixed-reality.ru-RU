# <a name="updating-2d-uwp-apps-for-mixed-reality"></a>Обновление 2D uwp для смешанной реальности

Windows Mixed Reality позволяет пользователю видеть голограммы, как если бы они непосредственно вокруг, в ваш физический и цифровой мир. По существу HoloLens и настольных ПК, присоединение иммерсивное гарнитура аксессуары, чтобы являются устройствами Windows 10; Это означает, что вы можете выполнять практически все приложения универсальной платформы Windows (UWP) в Store 2D приложений.

## <a name="creating-a-2d-uwp-app-for-mixed-reality"></a>Создание двухмерных приложения универсальной платформы Windows для смешанной реальности

Чтобы получить приложение работает как стандартное приложение 2D на ваш настольный монитор является первый шаг, чтобы разместить 2D приложение в гарнитуры смешанной реальности.

### <a name="building-a-new-2d-uwp-app"></a>Создание нового 2D приложения универсальной платформы Windows

Для создания нового приложения 2D для смешанной реальности, просто построить стандартный 2D приложения универсальной платформы Windows (UWP). Другие изменения приложения не являются обязательными для этого приложения для запуска в качестве баннер в смешанной реальности.

Чтобы приступить к сборке 2D приложения UWP, ознакомьтесь с [создайте свое первое приложение](https://docs.microsoft.com/windows/uwp/get-started/your-first-app) статьи.

Если уже имеется 2D приложения Windows в Store, сначала необходимо убедиться, что он предназначен для платформы Windows 10 Windows платформы Универсальной. Ниже приведены все потенциальные отправной точки, возможно, вместе с приложением Store уже сегодня.

Мы поговорим о том, что вы можете ограничить приложение специально для HoloLens, с помощью семейства устройств Windows.Holographic [ниже](#publish-and-maintain-your-universal-app).

### <a name="run-your-2d-app-in-a-windows-mixed-reality-immersive-headset"></a>Запустить приложение 2D в иммерсивных гарнитуры смешанной реальности Windows

Если вы развернули приложение 2D в настольном компьютере, где вы разрабатываете и проверил ее на мониторе, вы уже готовы опробовать ее в иммерсивных гарнитура рабочего стола!

Просто перейдите в меню "Пуск" в пределах гарнитуры смешанной реальности и запустите его оттуда. Оболочкой рабочего стола и holographic shell совместно использовать тот же набор приложений универсальной платформы Windows, и поэтому приложения, уже должны присутствовать после развертывания из Visual Studio.

## <a name="targeting-both-immersive-headsets-and-hololens"></a>Нацеливание иммерсивную и HoloLens

Поздравляем! Приложение теперь использует 10 универсальных Windows платформы Windows (UWP).

Приложение теперь может выполняться на современных устройств Windows, например рабочего стола, мобильных устройств, Xbox, Windows Mixed Reality иммерсивную и HoloLens, а также как будущих устройствах Windows. Тем не менее целевое расположение всех этих устройств, необходимо будет убедиться, что приложение предназначено для семейства устройств Windows.Universal.

### <a name="change-your-device-family-to-windowsuniversal"></a>Изменить Вашу семью устройства на Windows.Universal

Теперь перейдем к манифест AppX, чтобы гарантировать приложения универсальной платформы Windows для Windows 10 может быть запущен на HoloLens:
* Откройте файл решения приложения с **Visual Studio** и перейдите к манифест пакета приложения
* Щелкните правой кнопкой мыши **Package.appxmanifest** файлов в решении и перейти к **Просмотр кода**<br>
  ![Package.appxmanifest в обозревателе решений](images/openappxmanifest-500px.png)<br>
* Убедитесь, что целевой платформы Windows.Universal в разделе зависимостей
  ```
  <Dependencies>
    <TargetDeviceFamily Name="Windows.Universal" MinVersion="10.0.10240.0" MaxVersionTested="10.0.10586.0" />
  </Dependencies>
  ```
* Сохраните!

Если вы не используете Visual Studio для среды разработки, его можно открыть **AppXManifest.xml** в текстовом редакторе по своему усмотрению, чтобы обеспечить целевой платформой является **Windows.Universal**  *TargetDeviceFamily*.

### <a name="run-in-the-hololens-emulator"></a>Запуск в эмуляторе HoloLens

Теперь, когда приложение UWP предназначено для «Windows.Universal», давайте сборку приложения и запустите его [HoloLens эмулятор](using-the-hololens-emulator.md).
* Убедитесь, что у вас есть [эмулятор установлен, HoloLens](install-the-tools.md).
* В Visual Studio выберите **x86** сборки конфигурации для приложения

  ![x86 создать конфигурацию в Visual Studio](images/x86setting.png)<br>
* Выберите **HoloLens эмулятор** в раскрывающемся меню целевого развертывания

  ![Эмулятор HoloLens в список целей развертывания](images/deployemulator-500px.png)<br>
* Выберите **Отладка > Начать отладку** для развертывания приложения и запуска отладки.
* Эмулятор запустится и будет работать приложение.
* С помощью клавиатуры мыши и/или контроллер Xbox перевод приложения в мире, чтобы запустить его.

  ![Эмулятор HoloLens загружен образец универсальной платформы Windows](images/hololensemulatorwithuwpsample-800px.png)<br>

### <a name="next-steps"></a>Следующие шаги

На этом этапе может произойти следующее:

Это означает, что что-то пошло не так, и может занять несколько исследование, чтобы понять, как перенос приложения в жизнь в смешанной реальности.

Чтобы получить в нижнюю часть что способно являться причиной приложения универсальной платформы Windows не сможет запуститься на HoloLens, вы получите для отладки.

## <a name="update-your-ui"></a>Настройки пользовательского интерфейса

Теперь, когда приложения универсальной платформы Windows работает под управлением иммерсивную и/или HoloLens как двумерный голограмма, Далее мы будем убедитесь, что выглядит красивые. Ниже приведены некоторые соображения.
* Windows Mixed Reality будут работать все 2D приложения с фиксированным разрешением и точек на ДЮЙМ, которая указывает на 853 x 480 эффективный. Рекомендуется, если проект должен уточнения при таком масштабе и просмотрите руководство по проектированию, ниже, чтобы упростить работу HoloLens и иммерсивную.
* Windows Mixed Reality [не поддерживает](app-model.md) 2D живых плиток. Если ваши основные функциональные возможности сведения отображаются на живых иконках, рекомендуется переместить эту информацию обратно в приложение, или просмотреть [средствах запуска приложений 3D](3d-app-launcher-design-guidance.md).

### <a name="2d-app-view-resolution-and-scale-factor"></a>Коэффициент разрешения и масштаб представления 2D приложения

![Из адаптивный Дизайн](images/scale-500px.png)

Windows 10 перемещает все визуального проектирования из реальных экрана пикселей **эффективных пикселов**. Это означает, что разработчикам проектировать их пользовательского интерфейса после Windows 10 рекомендациям по интерфейсам для эффективных пикселов и масштабирования Windows обеспечивает эффективный пикселов правильного размера для удобства использования на устройствах, разрешения, DPI, и т.д. См. в разделе, это [отличная книга на сайте MSDN](https://msdn.microsoft.com/library/windows/apps/Dn958435.aspx) Дополнительные сведения, а также это [ПОСТРОЕНИЯ презентации](http://video.ch9.ms/sessions/build/2015/2-63_Build_2015_Windows_Scaling.pptx).

Даже с уникальные возможности для размещения приложений в ваш мир, в диапазоне расстояний расстояния стиле ТВ просмотра рекомендуется создание удобства и взаимодействия с взглядом/жестом. Из-за этого виртуального баннер на домашней странице реальности Mixed отображения неструктурированный представление UWP в:

**1280 x 720, 150% точек на ДЮЙМ** (эффективных пикселов 853 x 480)

Это разрешение имеет несколько преимуществ:
* Этот макет действующие пикселя будет иметь об одной информационной насыщенности как планшета или небольшого рабочего стола.
* Она соответствует фиксированного DPI и эффективных пикселов для приложений UWP, запущенных в службе Xbox One, включение возможность взаимодействия между устройствами.
* Этот размер вас устраивает в том случае, если масштаб наших различных операционных расстояния для приложений в мире.

HoloLens, сделает работу, чтобы убедиться, что приложение будет иметь шаблоны совместимых приложений, доступный для чтения текст размеров и соответствующий целевой попаданий изменения размера.
* Убедитесь в пользовательском Интерфейсе следующим советы и рекомендации по [гибкий Дизайн](https://msdn.microsoft.com/library/windows/apps/dn958435.aspx) для поиска, лучше всего подходит для HoloLen в уникальные разрешения и DPI.

**Не надо:**
* Слишком сильно изменяется пользовательский Интерфейс в смешанной реальности, чтобы убедиться, что пользователи имеют знакомый интерфейс и из него гарнитуры.

**Кнопка "Назад":** вызывает *[BackRequested](https://msdn.microsoft.com/library/windows/apps/windows.ui.core.systemnavigationmanager.backrequested.aspx)* событие при нажатии. Управляется назад видимость кнопки  *[SystemNavigationManager.AppViewBackButtonVisibility](https://msdn.microsoft.com/library/windows/apps/windows.ui.core.systemnavigationmanager.aspx)* .

## <a name="new-input-possibilities"></a>Новые возможности ввода

HoloLens использует расширенные глубины датчиков для см. в разделе всему миру, будут видеть пользователей. Это позволяет дополнительно привычные жесты, например [Блума](gestures.md#bloom) и [жест касания](gestures.md#air-tap). Мощные микрофоны также включить [голосовые возможности](voice-input.md).

С рабочего стола гарнитуры пользователи могут использовать контроллеры движения указывали на приложения и принимать меры. Они также могут использовать игровой, предназначенные для объектов с их взглядом.

Windows берет на себя все эти сложности для приложений UWP, преобразование вашей [помощи](gaze.md), жесты, голосовая связь и перемещения контроллера входных данных для [событий указателя](https://msdn.microsoft.com/library/windows/apps/mt404610#pointer_events) абстрагироваться от конкретных механизм ввода. Например пользователь может выполнить жест касания с их стороны или выберите триггер, полученная на контроллере движения, но 2D приложений не нужно знать, где входные данные поступили они просто см. в разделе 2D сенсорном нажатии кнопки "" как на сенсорном экране.

Ниже приведены высокого уровня основные понятия и сценарии, которые необходимо знать для входных данных при передаче приложения UWP в HoloLens.
* [Помощи](gaze.md) превращается в событий наведения указателя, которые могут неожиданно активировать, меню, всплывающие окна или других элементов пользовательского интерфейса должна всплывать достаточно gazing вокруг вашего приложения.
* Взглядом не с точностью до ввод от мыши. Используйте нужна для настройки размера попаданий целевых объектов для HoloLens, аналогичную мобильных приложений с поддержкой сенсорного ввода. Особенно трудно взаимодействовать с маленькими элементами на краях приложения:
* Пользователю нужно переключиться режимы ввода для перехода от прокрутки к перетаскивания для панорамирования два пальца. Если приложения был разработан для сенсорного ввода, рекомендуется, гарантируя, что за два пальца панорамирования заблокирована отсутствуют основные функциональные возможности. Если Да, рассмотрите возможность установки альтернативные способы ввода данных, таких как кнопки, которая может инициировать панорамирования два пальца. К примеру приложение карты можно увеличить с помощью двух пальцев панорамирования, но имеет знаком плюс, минус и кнопка для имитации же взаимодействий масштаб одной щелчками поворота.

[Голоса](voice-input.md) является важнейшей частью смешанной реальностью. Мы реализовали все интерфейсы API, которые в Windows 10, при использовании гарнитуры, включение Cortana речи.
