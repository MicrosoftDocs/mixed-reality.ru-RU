# <a name="direct-manipulation"></a>Прямые манипуляции
Прямого управления представляют собой входные модель, которая включает в себя касаясь голограммы прямо с рук. Цель с помощью прямого управления — объекты ведут себя так же, как и в реальном мире. Кнопки можно активировать, просто нажав их и объектов, могут быть получены, взяв их двухмерное содержимое ведет себя как виртуальный сенсорного экрана.  По этой причине прямые манипуляции прост для пользователей узнать, оно также прошло fun слишком.  Он считается «рядом с» модель ввода, это означает, что лучше всего подходит для взаимодействия с содержимым, которое находится в руки достичь.

Это ключевой, позволяет легко узнать прямого управления — это основе affordance. Существуют не символические жесты, чтобы обучить пользователей. Все взаимодействия должен быть построен вокруг визуальный элемент, который может быть затронутых или взял.

## <a name="collidable-fingertip"></a>Collidable срезах
На 2 HoloLens реальные руки пользователя распознаются и интерпретируются как левой и правой руки базовой модели. В идеале, чтобы реализовать идея касаясь голограммы прямо в руки, 5 colliders можно подключить к 5 распоряжение каждой стороны базовой модели. Однако на практике, из-за отсутствия tactile обратной связи, 10 collidable руках вызвать массу Непредвиденная и непредсказуемые конфликтов с голограммы. Таким образом мы советуем использовать только разместить collider на каждый вытянутым указательным пальцем. Коснитесь collidable индекс, который уже сейчас по-прежнему можно использовать в качестве активного сенсорного точки для различных сенсорные жесты, включающие другие соединители, такие как press палец 1, 1 палец, 2 нажмите пальца и нажмите клавишу 5 палец.

### <a name="sphere-collider"></a>Сфера collider
Вместо случайных обычную форму, мы рекомендуем использовать sphere collider и визуально их для предоставления подсказок лучше практически разрабатывать приложения для вывода. Диаметр сферы должен соответствовать толщина указательный палец для повышения точности распознавания сенсорного ввода. Его будет легко получить, вызвав API Рука переменной толщины палец.

### <a name="fingertip-cursor"></a>Срезах курсора
Дополнение к отображению collidable шар на срезах индекса, мы создадим решение advance, срезах курсор, чтобы добиться повышения рядом с выбора цели в интерактивном режиме. Это кольцевой курсора фигуры, прикрепить к срезах индекса. В соответствии с с учетом расположения он динамически реагирует на целевой объект в перспективе, ориентация и размер, как показано ниже:
* Если указательный палец перемещается к голограмма, курсор всегда параллельных на поверхность голограмма и постепенно уменьшается размер соответствующим образом. 
* Как только прикоснитесь палец, курсор сжимается в точку и генерирует событие сенсорного ввода.

<br> С помощью интерактивных обратной связи пользователей можно добиться высокой точности практически предназначенные для задач, таких как запуск гиперссылку на веб-содержимого или нажатии кнопки. <br>

## <a name="bounding-box-with-proximity-shader"></a>Ограничивающий прямоугольник с шейдера с учетом расположения
Голограмма, сам также требуется для предоставления отзывов visual и аудио компенсировать отсутствие tactile обратной связи. Для этого мы создаем концепцию ограничивающий прямоугольник с шейдера с учетом расположения. Ограничивающий прямоугольник имеет минимальный объемные область, включающую трехмерного объекта. Ограничивающий прямоугольник имеет механизм интерактивной подготовки к просмотру вызывается шейдера с учетом расположения. Выражение с учетом шейдера ведет себя следующим образом:

* Если указательный палец в пределах диапазона, spotlight срезах приводится на поверхности ограничивающий прямоугольник. 
* При получении срезах ближе к рабочей области, полезные сведения, объединяя соответствующим образом. 
* Как только прикоснитесь срезах, весь ограничивающий прямоугольник изменяет цвет или создать визуальный эффект для отражения состояния сенсорного ввода. 
* В то же время звуковой эффект может быть активирован для улучшения visual сенсорного.

![Ограничивающий прямоугольник с изображением шейдера с учетом расположения](images/Bounding-Box-With-Proximity-Shader-720px.jpg)<br>

## <a name="pressable-button"></a>Pressable кнопки
С помощью collidable срезах пользователи теперь готовы для взаимодействия с фундаментальнейшего holographic компонента пользовательского интерфейса, pressable кнопку. Pressable кнопки — это holographic кнопка, специально предназначенные для прямого палец press. Опять же, из-за отсутствия tactile обратной связи, pressable кнопки обеспечивает несколько механизмов для решения tactile отзывы проблемы, связанные с. 
* Первый механизм ограничивающий прямоугольник с шейдера с учетом расположения, который уже была устранена в вышеупомянутый абзац. Он служит для предоставления понятно, с учетом расположения для пользователей подходов и сделать контакта с кнопкой. 
* Второй — depression. Он создает смысле нажать после срезах обращается кнопки. Представляет собой механизм, что кнопка перемещается тесно с срезах вдоль оси глубины. Кнопки можно активировать как можно скорее достижения заданного глубины (на материалах издательства press) или оставить глубина (на выпуск) после передачи через него. 
* Звуковой должны быть добавлены для повышения обратной связи, в том случае, когда активирована кнопка.

## <a name="3d-object-manipulation"></a>Манипуляции трехмерного объекта
В 2 HoloLens пользователи могут использовать опускают руки для прямого управления объектами, 3D hologramphic, применяя к каждому трехмерному объекту ограничивающий прямоугольник. Ограничивающий прямоугольник предоставляет более глубокой через его шейдера с учетом расположения. С помощью ограничивающего прямоугольника существует два подхода к проектированию для манипуляции трехмерного объекта:      
### <a name="affordance-based-manipulation"></a>Affordance основе обработки:
Это позволяет пользователям управлять трехмерный объект через ограничивающий поле и читаемости манипуляции вокруг него. Как только руку пользователя близок к трехмерного объекта, ограничивающий прямоугольник и ближайшего affordance открываются. Пользователи могут получить ограничивающий прямоугольник для перемещения всего объекта читаемости edge для поворота и coner читаемости равномерно масштабировать.<br>

![Трехмерный объект обработки изображений](images/3D-Object-Manipulation-720px.jpg)<br>

### <a name="non-affordance-based-manipulation"></a>Non-affordance основе обработки:
В этом mechanisom не affordance присоединяется к ограничивающего прямоугольника. Пользователи могут только Показать ограничивающий прямоугольник, а затем напрямую взаимодействовать с ним. Если ограничивающий прямоугольник извлечен с одной стороны, перенос и поворот объекта связаны с движения и ориентации вручную. Когда объект извлечен с двумя руки, пользователям можно перевести, масштабировать и повернуть ее в соответствии с относительный движений две руки.<br><br> 

<br><br>
Для обработки требуется точность, мы рекомендуем afforance основе манипуляции, обеспечивая высокую степень гранулярности. Гибко управлять манипуляции не affordance будет хорошим выбором, предложить пользователям мгновенного и игровой.


## <a name="instinctual-gestures"></a>Instinctual жесты
В отличие от HoloLens (1-го поколения), несколько стандартных пользователей саду жесты, например Блума и Air, коснитесь HoloLens 2, мы не просим пользователям запоминать все символьные жест. Все жесты, необходимых пользователям для взаимодействия с голограммы и содержимое instinctual. Руководства пользователям выполнять жесты до разработки пользовательского интерфейса читаемости — подробное описание instinctual жест. Например если мы рекомендуем пользователям захвата объект или точку управления с жестом сжатия два пальца, объект или точку управления необходимо небольшой. Если мы предлагаем пользователям возможность выполнять пять захвата палец, объекта или контрольная точка должно быть сравнительно большой. Как кнопки, небольшая кнопка бы ограничить нажатия клавиш с одним пальцем, хотя огромные кнопки рекомендую пользователей нажать ее с их ладони.

## <a name="optimizing-with-eye-tracking"></a>Оптимизация с помощью отслеживания
Непосредственной работы со почувствуют магического, если он работает должным образом, но также быстро может стать утомительным, если не удается переместить свои силы в любом больше не непреднамеренно активируя голограмма.
Отслеживания потенциально помогает лучше определить, что такое намерения пользователя. 

* **Когда**: Уменьшите ошибочно активации обработки ответа. Отслеживания позволяет лучше понять, что пользователь настоящее время участвует с. Например представьте, что вы читаете holographic (инструкции) текст при достижении более, чтобы скопировать вам средства работы в реальных условиях.
Таким образом, вы случайно переместить свои силы на некоторые интерактивные holographic кнопки, вы даже не заметили перед (например, он даже за пределами пользователя поле зрения).
Короче говоря: Если пользователь еще не рассматривали голограмма какое-то время, но для него обнаружено событие сенсорного ввода или коммерческих тайн, весьма вероятно, что пользователь не был вы собираетесь фактически для взаимодействия с этой голограмма. 

* **Какой из них**: Помимо адресации false положительное количество активаций, еще один пример включает в себя лучше идентификации какие голограммы взять или выполнить поиск as точку пересечения точно не может быть ясны из вашей точки зрения, особенно в том случае, если несколько голограммы расположены рядом друг другие. Во время отслеживания глаза для HoloLens 2 имеет определенные ограничения о том, как точно выяснить глаз взглядом, это может по-прежнему быть весьма полезен для практически взаимодействия из-за глубины несоответствие при взаимодействии со стороны ввода. Это означает, что иногда трудно определить, является ли Рука или позади голограмма, чтобы точно скопировать манипуляции мини-приложение, например.

 * **Где**: Используйте сведения, что пользователь просматривает с быстрые жесты создает исключение. Захватите голограмма и примерно отбросить его к вашей пункта назначения. Хотя это иногда может работать нормально, быстро выполнять жестами руками может привести к высокой неточные назначения.
Это где отслеживания может помочь персонального Рука генерации вектор обратно к в нужное положение.
