# <a name="case-study-expanding-the-design-process-for-mixed-reality"></a>Пример: Расширение процесса разработки для смешанной реальности

Как корпорация Майкрософт выпустила HoloLens аудитории Безотложная разработчиков в 2016, группа уже привлечена studios внутри и вне корпорации Майкрософт позволяет создавать возможности запуска устройства. Эти команды, узнали во время работы, поиск исследованию возможностей и проблем в новом поле проектирования смешанной реальности.

Для новых рабочих групп и партнерам внедрять более эффективно, мы обращались свои методы и ценные сведения о следующей тематике проектирования и разработки занятий, которые мы обучение для разработчиков в нашей Academy реальности смешанный (включая семинары недельных разработки, мы предлагаем наши партнеры Enterprise).

Ниже приведен снимок эти знания частью более масштабного процесса, мы используем, чтобы помочь партнерам enterprise подготовить свои команды для смешанной реальности разработки. Хотя многие из этих методов изначально были затронуты для разработки HoloLens, ideating и обмена данными в трехмерном пространстве критически важны для полный спектр функций, смешанной реальности.

## <a name="thinking-spatially-during-the-design-process"></a>Мышление пространственно во время проектирования

Любой процесс проектирования или платформу разработки предназначен для выполнения итерации думать: Широко подходить к проблемы, чтобы эффективно обмениваться идеями с другими пользователями, для оценки этих идей и оптимального решения. В настоящее время у нас есть широко известных разработки и методы разработки для создания возможностей на ПК, телефоны и планшеты. Команды имеют четкое ожиданиям самое необходимое для итерации идею и включить его в продукт для пользователей.

Команды часто являются смесью разработки, проектирования, исследования и управления, но все роли могут участвовать в процессе проектирования. Препятствия, мешающие запись для передачи ему идею для мобильного приложения может быть простым, как нарисовать прямоугольник для экрана устройства. Хотя Создание эскиза полей или линий для элементов пользовательского интерфейса, с помощью стрелочки, обозначающие движения или взаимодействия, может быть достаточно, чтобы установить технических требований или определить потенциальные поведения пользователей.

Смешанной реальности, начинается процесс разработки традиционных 2D разделите: разметки в трехмерном пространстве сложно для большинства пользователей и средствами 2D, например пера и бумаги или досок, можно часто ограничивают идеи для их. В то же время инструменты 3D, построенный для игр или разработки, требующих высокого уровня квалификации, чтобы быстро детализировать идеи. Отсутствие облегченные средства осложняется технические неопределенности присущие с новые устройства, где методы фундаментальные взаимодействия по-прежнему устанавливаются. Эти проблемы потенциально можно ограничить вклад проектирования рабочей группы могут выполнять только те с фоном 3D-разработки, существенно сокращая способность группы к итерации.

![Команды, работающие во время семинар в академии реальности Mixed](images/academyTeam1000.png)<br>
*Команд со смешанной реальности партнерской программе в нашей семинар*

При работе с внешними партнерами, мы слышим истории команд «ожидание разработчику окончание прототип» прежде чем их можно будет продолжить процесс разработки. Часто это означает, что остальная часть команды блокируется в осмысленные идет работа над продуктом, перегрузка разработчика для решения технической реализации как основные компоненты пользователя (поскольку они пытаются стоечный примерное представление кода).

## <a name="techniques-for-expanding-the-design-process"></a>Методы для расширения процесса разработки

Нашими командами разработчиков в корпорации Майкрософт имеют набор методов для более эффективного включают их группе и быстрой итерации разработки сложных проблем. Во время формального процесса эти методы предназначены для дополнения, а не заменяют рабочего процесса. Эти методы позволяют без специальные навыки 3D предложить идеи, прежде чем углубляться в на этапе создания прототипов. Различные роли и дисциплины (не только 3D конструкторы и разработчики) может быть частью процесса разработки, чтобы извлекать возможные сделки и определение возможных проблем, которые могут возникнуть позднее в разработке.

## <a name="generating-ideas-with-bodystorming"></a>Формирование идеи с bodystorming

Получение поддержки, чтобы думать о событиях, происходящих в реальном мире, помимо традиционных мире двухмерной устройств является ключом к разработке инновационных смешанной реальности возможности. В корпорации Майкрософт мы обнаружили, что лучший способ сделать это — Чтобы стимулировать взаимодействие с физической props в реальных условиях пространстве. С помощью простого и недорогого создавая материалов мы создаем физических свойств для представления цифровые объекты, пользовательские интерфейсы и анимации в предложенный раньше. Этот прием называется bodystorming и стал стандартом идеальная продукта дизайна, промышленный десятилетиями.

![Команды, работающие с материалами в академии реальности Mixed](images/academyWork1000.png)
*простого и недорогого art источники, используемые в bodystorming*

Простой, физических свойств уровня игрового поля для участников, отдельных сотрудников с овладевать различными и фоном, добавлять идеи и откройте для себя возможности встроенные компоненты для работы с смешанной реальности Избавьтесь парадигму 2D мышление. Хотя технические создания прототипов или высокой точностью воспроизведения раскадровки требует опытный разработчик 3D или исполнителя, несколько Styrofoam шары и картон может быть достаточно, чтобы продемонстрировать, каким образом интерфейс может раскрыть на физическом пространстве. Эти методы применяются для смешанной реальности разработки с HoloLens и иммерсивную. Например набор пластиковыми соединители примерно можно проиллюстрировать размер голограммы, которые отображаются в интерфейсе HoloLens, так и props действовать элементом элементы или макеты движения в виртуальном мире.

Bodystorming — это метод быстро создать идеи и оценки идей, которые являются слишком гипотетической прототип. В корпорации Microsoft bodystorming чаще всего используется для быстрого тестирования идею, несмотря на то, что может быть полезным для размещения углубленные сеанса, если задействовать внешних заинтересованных лиц, которым не знакомы с разработкой смешанной реальности или необходимые для преобразования очень широкий круг вопросов сценариев. Помните, что bodystorming предназначена для ideate и эффективно поощряя участников пространственно подумать. Подробные рисунки или точно не важны на этом этапе. Физические свойства требуется только соответствуют минимальным требованиям для просмотра или взаимодействия идею. Идеи, представленную с помощью bodystorming не должны быть полностью проверены, но процесс может помочь сузить возможностей для тестирования впоследствии на этапе создания прототипов в устройство. Таким образом bodystorming не предназначен для технических создания прототипов, замените, но вместо смещение нагрузку решения обоих technical и проектирования проблем на этапе создания прототипов.

## <a name="acting-and-expert-feedback"></a>Отзыв действующего и экспертов

Следующий процесс bodystorming ideating с физическим объектам в реальном мире следующий шаг — предоставление поэтапного посредством интерфейса с этими объектами. Мы называем этот этап процесса действующего и обычно применяются размещения, как пользователь перемещает запуске, или определенного взаимодействия.

![Команды, выполняющий сценарий развертывания во время семинар в академии реальности Mixed](images/academyCollab1000.png)<br>
*Команды, выполняющий сценарий развертывания во время обсуждения*

Работе с физической props позволит участникам не только возможности обдумывание точки зрения пользователя, но разрешить внешним наблюдателям см. в разделе, как события воспроизводиться. Это представляет идеальным дополнением для включения более широкого круга членам группы и заинтересованных лиц, которым можно предоставить отзыв «экспертов». Например если вы исследуете смешанной реальности, удобство для больницы, выполняющий out мышление со специалистом медицинских отзывов бесценным. Другой пример — в том, что отдельная проблема, которую вы пытаетесь понять, как пространственных аудио- или бюджет для ресурса качества и производительности. В этих случаях действующего дает экспертов быстро, примерно концепцию как процесс может раскрыть, без необходимости прототипа на основе устройств.

Такого рода действующего не формализованы, не нужно перевести в профессиональных кадров, но в ситуациях, где мы хотим поделиться с достижениями с другими пользователями, не являющиеся отсутствует, мы выполним видео записи «сцены» взаимодействия или включать исполнителя раскадровки, наблюдать и нарисуйте ключ секунд. Действующего также может быть очень простая действия, часто происходит на месте во время фазы bodystorming. Можно использовать, зависит от аудитории (и точность, необходимые для выявления необходимых тип отзыва), но в конечном счете сводится к все, что наиболее эффективно выполнит захват мышления вашей команды.

## <a name="capturing-ideas-with-storyboards"></a>Воплощения идей с помощью Storyboard

Наилучший способ передачи идей и основные понятия работы предложенное зависит от предполагаемой аудитории, а также тип обратной связи, которые требуются вашей следующей итерации. При представлении новые идеи, членам команды, повторной enactments низкого качества из bodystorming может быть достаточно, чтобы привести работы с этой системой. При внедрении вашей концепции качества новый заинтересованных лиц или потенциальных пользователей, лучший способ часто раскадровки. Раскадровка — это методика, общие для индустрии развлечений, обычно находится в фоновом в фильмах и Разработка компьютерных игр, и помогает передачи обоих общий поток работы (с низкой fidelities), а также эстетически внешнего вида и поведения (с высокой fidelities). Как и для создания прототипов, понимания потребностей fidelity раскадровки критически важна для сбора нужного вида обратной связи и предотвращения нецелесообразными обсуждения.

![Пример раскадровки низкой четкости](images/storyboardLow1000.png)<br>
*Пример раскадровки низкой четкости*

Меньшей точностью воспроизведения раскадровки представляют собой правой точность для быстрого обсуждения, особенно в том случае, если передача высокоуровневых идей. Это могут быть сложнее, чем схематичное рисунков и простые фигуры для обозначения виртуальные элементы в сцене или расстоянию между Интерактивные компоненты (физических и виртуальных). Хотя эти функции удобны, заданному относительно легко и низкой навыков препятствием для выполнения, как вы помните урока bodystorming: Не у всех имеются 2D сцены работы и понимать последствия 3D.

![Пример высокой точностью воспроизведения раскадровки](images/storyboardHigh1000.png)<br>
*Пример высокой точностью воспроизведения раскадровки*

Высокоточную раскадровки представляют собой мощное средство, при переводе в новый заинтересованных лиц, или при объединении ценной информации из сеанса bodystorming с предложенное эстетически направление работы. Создание раскадровки с использованием можно доски настроения проиллюстрировать окончательного вида виртуальный интерфейс, а также для захвата ключевые моменты, которые могут быть pivotal к окончательной версии продукта. Имейте в виду, что высокой точностью воспроизведения раскадровки часто требуют исполнителя, особенно один внедрен в группе, можно фиксировать сложно описывают идеи. В корпорации Майкрософт мы добавили исполнители раскадровки, часто с фоном в разработке игр, наши команды смешанной реальности, посещать встречи, готовы быстро получить эскизы, которые позже будут расширены в более поздней версии точность модели. Эти пользователи тесно сотрудничать с дизайнеров, помогая передачи художественного направление ресурсы, используемые в окончательной версии.

## <a name="expanding-your-teams-skills"></a>Расширение навыков вашей команды

Помимо описанных, изменение процесса можно подобрать для смешанной реальности возможности зависит от более тесно подключение технические и конструктора ролей. Работа с партнерами корпорации Майкрософт, мы обнаружили несколько групп успешно перейти в 3D-разработки и найти наибольшим преимуществом поступает от членов группы, шаг с выходом из зоны комфорта и развитие навыков, которые предоставляют им больше труда в процессе разработки (не только ограничение свои навыки с проектирования и разработки).

Итерации и обратной связи является ключом к любой успешной разработки, а в смешанной реальности опыт такого рода отзыв часто Наклоняет на более подробные технические понятия, особенно учитывая относительный nascence технологий и средств смешанной реальности. Навыков проекту группы в технических областях, включая профессиональное знание средств, таких как Unity и Unreal, помогает пользователям, лучшего понимания принципов работы разработчиков и более эффективно отзыв во время создания прототипов. Аналогичным образом разработчики, понимаете фундаментальные концепции UX опыт смешанной реальности (и потенциальные ошибки, связанные с разработки) может помочь их предлагают insights реализации на ранних этапах планирования.

Обычно мы получаем вопросы от партнеров в смешанной реальности, о том, как лучше всего работает в их группе расти и навыки с помощью средств 3D. Это вопрос сложно ответить этой отрасли, так как команды все чаще сталкиваются с создания будущих возможностей с инструменты вчерашнего дня. Многие команды сбор отдельных пользователей с фоном в игры и развлечения, где 3D разработки стал стандартом десятилетиями или поощряя их существующих команд, чтобы забрать средств, таких как Unity и 3D-моделирования программного обеспечения. Хотя это и оба потребности по соблюдению базовые возможности смешанной реальности сегодня, очень вероятно, будут огромное количество специализированные средства и навыки для создания приложений революционного в завтра.

Поле проектирования смешанной реальности значительно изменится в ближайшее время, с помощью сконцентрироваться на данных и облачных служб в holographic опыт. Этот следующего поколения приложений будет воспользоваться преимуществами расширенной естественного языка и компьютерного зрения в режиме реального времени, а также по своей природе более социальных сетей сценариев в Впечатляющие возможности. Например интерфейсов в виртуальной реальности разработчики для создания творческих сред и пробелы, требующие навыков больше похоже архитектуры или психологии, а не традиционным интерфейсом разработки. Рассмотрите возможность работы в реальном мире с HoloLens, где случаи часто включают специализированной поля, такие как медицина, проектирование, или производство, где определенных средах и в режиме реального времени данные являются важными элементами интерфейса. Работа по ролям, используя проектирования и специальных знаний и продолжение готовность к постоянному обучению новые средства являются бесценными навык для команды, работающие в смешанной реальности.

## <a name="helping-your-team-explore-mixed-reality-quickly-and-effectively"></a>Помочь вашей группе изучение смешанной реальности, быстро и эффективно

В первых дней существования HoloLens, эти методы были рассылкой по необходимости могут как устройство ранний прототип оборудования стал меньше, чем идеально формат для быстрого прохода своих идей по разработке. Методы взаимодействия, включая основные пользовательского интерфейса в операционной системе были использовать bodystorming, пока действует помогли наша группа удобство использования понять поведение основных пользователей для сценариев, в рабочей области. В конце концов, группа смогла определить строгое базовые показатели для core UX для HoloLens, блокируя понятия ролевых кластеров, позволяя операционной системы разработки для быстрого перемещения параллельно с разработкой оборудования.

![Семинар материалов в академии реальности Mixed](images/academyMat1000.png)<br>

Будь то bodystorming, выполняющий, проверки с экспертами и раскадровки, эти методы предназначены для сэкономить время и усилия. Хотя мы по-прежнему очень много, в первых дней существования HoloLens и разработки виртуальной реальности, развернув процесса проектирования для смешанной реальности команда сможет тратить их энергии, исследование проблем проектирования новых и сложной задачей, а не преодолению трудностей взаимодействие идеи.
