# <a name="mr-learning-base-module---lunar-module-assembly-sample-experience"></a>Модуль MR обучения Base - качества образец модуля лунного сборки

На этом занятии мы объединит несколько понятий, узнали из предыдущих занятий, для создания уникальных пример взаимодействия. Мы создадим приложение сборки лунного модуля, при котором пользователю будет необходимо использовать отслеживаемые руки для применения модуля лунного частей и попытаться собрать лунного модуля. Мы будем использовать pressable кнопки для переключения размещения подсказки, чтобы сбросить опыт и запуск нашего модуля лунного в пространство! В последующих руководствах мы будем продолжать созданы на основе этой работы, включая мощные несколькими пользователями варианты использования, которые использует Azure пространственных привязки для выравнивания пространственных.

## <a name="objectives"></a>Цели

- Объединение нескольких концепции из предыдущих занятий, чтобы создать уникальное взаимодействие
- Узнайте, как переключить объекты
- Сложные события триггера с помощью pressable кнопок
- Использование элемента rigidbody физики и силы
- Изучить использование всплывающих подсказок

## <a name="instructions"></a>Инструкция

### <a name="configuring-the-lunar-module"></a>Настройка модуля лунного

В этой главе мы будут представлены различные компоненты, необходимые для создания опыт образец.

1. Добавление готового блока сборки модуля лунного в сцену базы. Для этого на вкладке проект, найдите «Launcher_Tutorial Rocket». Можно также найти в ресурсах prefab > BaseModuleAssets > Prefabs. Может появиться два prefabs rocket запуска; один с именем «учебник», а другой с именем «выполнено». Перетащите «Rocket Launcher_Tutorial» prefab в сцену базы. Вы можете разместить размещение prefab в сцене.
   Примечание. Prefab «Launcher_Complete Rocket» является завершенной указано средство запуска, для ссылки.

Материал «рентгеновский снимок» позволяет немного полупрозрачный цвет, который мы будем использовать в качестве размещения подсказки для пользователя.

### <a name="lunar-module-parts-overview"></a>Общие сведения о частях лунного модуля
Модуль лунного частей родительского объекта — коллекция объектов, с которыми будет взаимодействовать пользователь. В списке ниже приведены имена игровому объекту (с сцены с меткой имена в paretheses):

- Рюкзаке (топливного)
- GasTank (ячейка энергии)
- TopLeftBody (Rover корпус)
- А нос — папы (закрепление портал)
- LeftTwirler (внешний датчик)

Обратите внимание на то, что каждый из этих объектов имеет обработчик манипуляции, как обсуждалось на занятии 4. С помощью обработчика манипуляции пользователи могут взять и манипулировать объектом. Также Обратите внимание, что параметр «два типа выдан манипуляции» имеет значение «перемещать и поворачивать.» Это позволяет пользователю перемещать объект и не изменять свой размер, являющийся нужной функции для сборки приложения.
Кроме того дальней манипуляции не установлен, чтобы разрешить только для прямого взаимодействия частей модуля.

![Lesson6 Chapter2im](images/Lesson6_Chapter2im.PNG)

Демонстрационный сценарий часть сборки (как показано выше) является сценария, который управляет объектами помещается в модуле лунного пользователем. 

Поле «Объект на месте» является преобразование, которое выбранный (н регистр на рисунке выше, рюкзаки/топливного) с объектом, который он может подключаться к. 

Параметры «Рядом с Distance» и «Далеко Distance» несете ответственность за определение с учетом расположения, к которому частей страничным на месте или выпуск. Например рюкзаки/топливного потребуется 0,1 единицы от модуля лунного встать на свое место. «Далеко Distance» задает расположение, где должен быть отсоединиться от модуля лунного объекта. В этом случае руку пользователя необходимо взять рюкзаке/топливного и потяните за нее 0,2 единицы от модуля лунного, чтобы удалить его из обратно к месту.

«Объект подсказки средства» — это метка совет средство в сцене. Если объекты имеют почасовую привязку на месте, метки будут отключены. 

Источник аудио будет извлечен автоматически. 

### <a name="placement-hints-buttons"></a>Размещение указания кнопок
В [Урок 2](mrlearning-base-ch2.md), вы узнали, как размещение и настройка кнопки для работы, например изменение цвета элемента или сделать его воспроизведения звука при его отправке. Мы будем продолжать использовать эти принципы, как мы настроим наши кнопки переключения размещения подсказки. 

Целью является настройте кнопку таким образом, чтобы каждый раз при нажатии кнопка Указание размещения, он будет переключать видимость прозрачные размещения подсказок. 

Шаг 1. Переместите модуль лунного слот пустой «только среда выполнения» на панели инспектора, пока объект размещения подсказки выбран в иерархии базовых сцены. 
 ![Lesson6 Chapter3 Step1im](images/Lesson6_Chapter3_step1im.PNG) шаг 2: Теперь щелкните раскрывающийся список, где отображается надпись, «нет функция». Перейдите «TogglePlacementHints» и в этом меню выберите «ToggleGameObjects ().» ToggleGameObjects() будет включать размещения подсказки и выключать, чтобы они были видимым или невидимым, каждый раз при нажатии кнопки.
 ![Lesson6 Chapter3 Step2im](images/Lesson6_Chapter3_step2im.PNG)

### <a name="configuring-the-reset-button"></a>Настройка кнопки сброса

Будет существовать ситуации, когда пользователь делает ошибку, или случайно создает исключение, объект немедленно, или просто хочет rest в интерфейсе. «Сброс» будет добавлена возможность перезапустить процесс. 

Шаг 1. Нажмите кнопку сброса. В базовый сцены он называется «ResetRoundButton.» 

Шаг 2. Перетащите модуль лунного из иерархии базовых сцены в пустой слот, в разделе «нажата кнопка» на панели инспектора.
 ![Lesson6 Chapter4 Step2im](images/Lesson6_Chapter4_step2im.PNG)

Шаг 3. Выберите в раскрывающемся меню, который говорит: «нет функции» и наведите указатель мыши «LaunchLunarModule.» Теперь выберите «resetModule ()».

![Lesson6 Chapter4 Step3im](images/Lesson6_Chapter4_step3im.PNG)

> Примечание. Обратите внимание, что по умолчанию «GameObject.BroadcastMessage» настроен для «ResetPlacement.» Это будет широковещательная передача сообщения о вызове «ResetPlacement» для всех дочерних объектов RocketLauncher_Tutorial. Любой объект, который содержит метод для «ResetPlacement()» будет отвечать на это сообщение, сбросив его позиции. 

### <a name="launching-the-lunar-module"></a>Запуск модуля лунного
В этой главе мы будет выполняться настройка кнопку запуска. Это позволит пользователю нажать кнопку и запустить модуль лунного в пространство.

Шаг 1. Выберите кнопку запуска (в базовый сцене он называется «LaunchRoundButton»). Перетащите модуль лунного пустой слот, в разделе «End Touch» на панели инспектора.
 ![Lesson6 Chapter5 Step1im](images/Lesson6_Chapter5_step1im.PNG) шаг 2: Выберите в раскрывающемся меню с сообщением «не функция». Наведите указатель мыши «LaunchLunarModule» и выберите «StopThruster ()». Это будет контролировать объем Осевая сила, пользователю необходимо передать лунного модуля. 
 ![Lesson6 Chapter5 Step2im](images/Lesson6_Chapter5_step2im.PNG) шаг 3: В разделе «ButtonPressed()» добавьте модуль лунного (, удержание, перетаскивание) в пустой слот. 

Шаг 4. Щелкните раскрывающееся меню с сообщением «нет функции» и наведите указатель мыши «LaunchLunarModule» и выберите «StartThruster ()». 
 ![Lesson6 Chapter5 Step4im](images/Lesson6_Chapter5_step4im.PNG) шаг 5: Добавьте модуль лунного музыки таким образом, чтобы при ракеты набирает обороты, будет воспроизведение музыки. Для этого перетащите модуль лунного Далее пустой слот, в разделе «Pressed() кнопка».

Шаг 6. Выберите в раскрывающемся меню, который говорит: «нет функции» и наведите указатель мыши «AudioSource» и выберите «PlayOneShot (AudioClip)». Вы можете просмотреть все разнообразие звуков, в состав MRTK. В этом примере мы будем придерживаться «MRTK_Gem.»
 ![Lesson6 Chapter5 Step6im](images/Lesson6_Chapter5_step6im.PNG)
