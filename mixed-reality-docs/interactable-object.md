# <a name="interactable-object"></a>Взаимодействующий объект

Кнопка длиннее метафоры, используемой для активации события в 2D-абстрактном мире. В мире объемной смешанной реальности мы больше не должны беспокоиться об этом мире абстракции. Любой может быть взаимодействующим **объектом** , запускающим событие. Взаимодействующий объект может быть представлен любым из чашк кофе в таблице в виде всплывающей подсказки, плавающей в воздухе. Мы по-прежнему используем традиционные кнопки в определенных ситуациях, например в пользовательском интерфейсе диалогового окна. Визуальное представление кнопки зависит от контекста.

![Объекты интерактибле](images/640px-interactibleobject-hero-640px.jpg)


## <a name="important-properties-of-the-interactable-object"></a>Важные свойства взаимодействующего объекта

### <a name="visual-cue"></a>Визуальная подсказка

Визуальные подсказки — это подсистемы датчиков, получаемые глазом в виде освещения и обрабатываемые визуальной системой во время визуального восприятия. Так как визуальная система находится во многих виды цветов, особенно для людей, визуальные подсказки представляют собой большой источник информации о том, как воспринимается мир.

В смешанной реальности, поскольку holographic объектов смешана с реальной средой, может быть трудно понять, какие объекты являются взаимодействующими. Для любых взаимодействующих объектов важно обеспечить дифференцированное визуальное подсказку для каждого входящего состояния. Это помогает пользователю понять, какая часть вашего интерфейса является взаимодействующей, и делает пользователя надежным благодаря согласованному методу взаимодействия.

#### <a name="far-interactions"></a>Дальнее взаимодействие

Для любых объектов, с которыми пользователь может взаимодействовать с помощью элемента управления "взгляд, рука и луч контроллера движения", рекомендуется иметь разные визуальные подсказки для этих трех состояний входа:
* **По умолчанию (наблюдение)** : Состояние простоя объекта по умолчанию.
* **Целевой (наведение)** : Когда объект ориентирован на курсор «взгляд», указатель мыши на палец или контроллер движения.
* **Нажато**: Когда объект нажимается с помощью жеста касания, нажмите кнопку "выбрать" для пальца или контроллера движения.

Вы можете использовать такие методы, как выделение или масштабирование, чтобы обеспечить визуальную подсказку для входных состояний пользователя. В Windows Mixed Reality можно найти примеры визуализации различных входных состояний в меню «Пуск» и на кнопках панели приложений.

#### <a name="neardirect-interactions"></a>Близкие (прямые) взаимодействия

HoloLens 2 поддерживает вводные данные отслеживания, которые позволяют взаимодействовать с объектами. Без хаптик отзывов и более глубокого восприятия иногда может быть трудно определить, насколько далеко от руки происходит из объекта, а также от того, соприкасаются ли вы. Очень важно предоставить достаточно визуальных подсказок для передачи состояния объекта и в частности, в отношении голограмм.

Используйте визуальную обратную связь, чтобы сообщить следующее:
* **По умолчанию (наблюдение)** : Состояние простоя объекта по умолчанию.
* **Наведение**: Когда руку приближается к голограмме, измените визуальные элементы так, чтобы они могли взаимодействовать с голограммами. 
* **Расстояние и точка взаимодействия**: По мере приближения к голограммам, разработка отзывов, чтобы сообщить о предполагаемой точке взаимодействия, а также расстояние от объекта, на котором наследует палец
* **Начало контакта**: Измените визуальные элементы (светло, Color), чтобы сообщить о возникновении этой сенсорной связи
* За: Изменение визуальных элементов (светлого, цветного) при проходе этого объекта.
* **Окончание контакта**: Изменение визуальных элементов (светлого, цветного) при завершении сенсорного ввода.

### <a name="audio-cue"></a>Звуковые подсказки
Для взаимодействия непосредственно с руки правильная обратная связь может значительно улучшить взаимодействие с пользователем. Используйте отзыв о звуке, чтобы сообщить следующее:
* **Начало контакта**: Воспроизводить звук при начале сенсорного ввода
* **Окончание контакта**: Воспроизвести звук на сенсорном конце
* **Начало захвата**: Воспроизводить звук при начале захвата
* **Конец захвата**: Воспроизводить звук на стороне захвата

### <a name="voice-command"></a>Voice, команда
Для любых взаимодействующих объектов важно поддерживать альтернативные варианты взаимодействия. По умолчанию рекомендуется поддерживать команду Voice для всех объектов, которые являются взаимодействующими. Чтобы улучшить возможность обнаружения, можно предоставить подсказку о состоянии наведения.

## <a name="sizing"></a>Изменения размера
Чтобы пользователи могли легко затронуть все взаимодействующие объекты, рекомендуется убедиться, что они соответствуют минимальному размеру (визуальный угол, часто измеряемый в градусах визуальной дуги) в зависимости от расстояния, которое он помещает от пользователя. Визуальный угол основан на расстоянии между глазами пользователя и объектом и остается постоянным, а физический размер целевого объекта может измениться по мере изменения расстояния от пользователя. Чтобы определить необходимый физический размер объекта на основе расстояния от пользователя, попробуйте использовать визуальный калькулятор угла, такой [как.](http://elvers.us/perception/visualAngle/)

* [Элементом](https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/README_Interactable.html)
* [Button](https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/README_Button.html)
* [Сцены с примерами взаимодействия руки](https://github.com/microsoft/MixedRealityToolkit-Unity/blob/mrtk_release/Documentation/README_HandInteractionExamples.md)

Стандартный шейдер Микседреалититулкит предоставляет различные параметры, такие как **освещение** , позволяющее создавать визуальные и звуковые подсказки.
