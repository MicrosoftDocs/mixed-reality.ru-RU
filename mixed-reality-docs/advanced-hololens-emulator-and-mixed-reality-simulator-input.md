# <a name="advanced-hololens-emulator-and-mixed-reality-simulator-input"></a>Дополнительные входные данные эмулятора HoloLens и смешанной реальности симулятора

Большинство пользователей эмулятор будет достаточно использовать основные элементы управления вводом для [эмулятор HoloLens](using-the-hololens-emulator.md#basic-emulator-input) или [симулятор Windows Mixed Reality](using-the-windows-mixed-reality-simulator.md#basic-simulator-input). Ниже приведены для опытных пользователей, пришли к выводу необходимость имитировать более сложные типы входных данных.


## <a name="concepts"></a>Понятия

Чтобы начать работу, управление виртуальный вход в симуляторе, эмуляторе HoloLens и Windows Mixed Reality, необходимо сначала изучить несколько понятия.

Движения ссылается на управление и изменив положение и ориентацию чего-либо в сцене. Для целевого объекта управляемой движения контролируется смену и переноса (движения) по трем осям.
* **Поворот**: Включение влево или вправо.
* **Шаг**: Включите вверх или вниз.
* **Откатить**: Восстановить в сторону.
* **X**: Переместите влево или вправо.
* **Y**: Перемещение вверх или вниз.
* **Z**: Переместите вперед или назад.

[Жест](gestures.md) и входные данные контроллера движения тесно сопоставляются как они физические устройства:
* **Действие**: Эта процедура имитирует действия, нажав клавишу указательным пальцем на бегунок или извлечение кнопку действия на контроллере. Например входные данные действия можно использовать для имитации жест жест касания, выполнять прокрутку содержимого и на нажатие и удерживание.
* **[Блума](gestures.md#bloom)/System жест или домашнего**: Жест bloom/system HoloLens или кнопку Home контроллер используется для возврата в оболочку и для выполнения действий системы.

Руки имеют широкие возможности reprresentation в HoloLens 2.  Кроме того, что отслеживаемые или не распознан и может использоваться для вождения жесты, руки теперь имеет ясно сформулированные скелет модели их по размеру и открывается разработчику.  Это приводит к 26 отслеживаемые точек с каждой стороны.  
* **Совместные**: Одно из двадцати отслеживаемые позиции для заданной отслеживаемой руки. Это свойство будет иметь точку является трехмерном пространстве, связанные с ней.
* **Представлять**: Полный коллекцию всех составов отслеживаемые руки. В настоящее время это коллекция 26 шарнирные соединения. 

В настоящее время мы не предоставляем непосредственным управлением Каждая позиция совместные отдельно с помощью пользовательском интерфейсе эмулятора на то, что их можно установить через имитации API. Вместо этого у нас есть набор полезных репрезентативной создает, эмулятор позволяет переключаться между.

Можно также управлять состояние ввода имитируемого датчика:
* **Сброс**: Эта команда возвращает всех имитированных датчиков к значениям по умолчанию.  Начиная с эмулятором 2 HoloLens, сброс могут задаваться для одного или обоих руки требуемой hand(s), используя соответствующий модификатор ключи или button(s) (слева и/или Alt правой или левой или правой прокладкой на игровой), проводимых корпорацией.
* **Отслеживание**: Циклическое переключение между режимами позиционные отслеживания. Сюда входят следующие возможности.
  * **По умолчанию**. Операционная система выбирает оптимальный режим отслеживания на основе запросов системы.
   * **Ориентация**: Заставляет только для ориентации отслеживания, независимо от запросов системы.
   * **Позиционные**: Заставляет позиционные отслеживания, независимо от запросов системы.

## <a name="perception-simulation-control-panel-keyboard-shortcuts"></a>Восприятие панели управления моделирования сочетания клавиш

Доступны следующие сочетания клавиш для доступа к панели управления моделирования восприятие и включение или отключение устройств ввода ПК для использования с моделирования.

| Операция | Сочетание клавиш | Описание и примечания |
|-----------|----------|-------------|
| Переключить «Использовать клавиатуру для моделирования» | F4 | При выключенном ввод с клавиатуры переходит к приложению HoloLens или смешанной реальности Windows. |
| Переключить «Использовать мышь для моделирования» | F5 | При выключенном ввод от мыши поступает в смешанной реальности среду (Windows Mixed Reality только) |
| Переключить «Использовать игровой для моделирования» | F6 | При выключенном игровой ввод игнорируется, моделирование |
| Отображение или скрытие панели управления | F7 | |
| Установить фокус на панель управления | F8 | Если панель не видимую в данный момент он будут показаны первыми. |
| Закрепить или открепить панели и из эмулятора или окно портала смешанной реальности | F9 | Если окно закрыто при открепить, закреплено и скрыты. |
