# <a name="advanced-hololens-emulator-and-mixed-reality-simulator-input"></a>Дополнительные входные данные эмулятора HoloLens и смешанной реальности симулятора

Большинство пользователей эмулятор будет достаточно использовать основные элементы управления вводом для [эмулятор HoloLens](using-the-hololens-emulator.md#basic-emulator-input) или [симулятор Windows Mixed Reality](using-the-windows-mixed-reality-simulator.md#basic-simulator-input). Ниже приведены для опытных пользователей, пришли к выводу необходимость имитировать более сложные типы входных данных.

> [!NOTE]
> Дополнительные рекомендации, относящиеся к HoloLens 2 [ожидается в ближайшее время](index.md#news-and-notes).

## <a name="concepts"></a>Понятия

Чтобы начать работу, управление виртуальной входных данных для эмулятора HoloLens и симулятор Windows Mixed Reality, необходимо сначала изучить несколько понятия.

Движения ссылается на управление и изменив положение и ориентацию чего-либо в сцене. Для целевого объекта управляемой движения контролируется смену и переноса (движения) по трем осям.
* **Поворот**: Включение влево или вправо.
* **Шаг**: Включите вверх или вниз.
* **Откатить**: Восстановить в сторону.
* **X**: Переместите влево или вправо.
* **Y**: Перемещение вверх или вниз.
* **Z**: Переместите вперед или назад.

[Жест](gestures.md) и входные данные контроллера движения тесно сопоставляются как они физические устройства:
* **Действие**: Эта процедура имитирует действия, нажав клавишу указательным пальцем на бегунок или извлечение кнопку действия на контроллере. Например входные данные действия можно использовать для имитации жест жест касания, выполнять прокрутку содержимого и на нажатие и удерживание.
* **[Блума](gestures.md#bloom) или Главная**: HoloLens цвести жест или когда контроллера Home кнопка используется для возврата в оболочку и для выполнения действий системы.

Руки имеют широкие возможности reprresentation в версии 2 HoloLens.  Кроме того, что отслеживаемые или не распознан и может использоваться для вождения жесты, руки теперь имеет ясно сформулированные скелет модели их по размеру и открывается разработчику.  Это приводит к 20 точек, отслеживаемых с каждой стороны.  
* **Совместные**: Одно из двадцати отслеживаемые позиции для заданной отслеживаемой руки. Это свойство будет иметь точку является трехмерном пространстве, связанные с ней.
* **Представлять**: Полный коллекцию всех составов отслеживаемые руки. В настоящее время это коллекция 20 соединений. 

В настоящее время мы не предоставляем непосредственным управлением Каждая позиция совместные отдельно с помощью пользовательский Интерфейс эмулятора. Однако можно установить их через имитации API. Вместо этого у нас есть набор полезных репрезентативной создает, эмулятор позволяет переключаться между.

Можно также управлять состояние ввода имитируемого датчика:
* **Сброс**: Эта команда возвращает всех имитированных датчиков к значениям по умолчанию.
* **Отслеживание**: Циклическое переключение между режимами позиционные отслеживания. Сюда входят следующие возможности.
  * **По умолчанию**. Операционная система выбирает оптимальный режим отслеживания на основе запросов системы.
   * **Ориентация**: Заставляет только для ориентации отслеживания, независимо от запросов системы.
   * **Позиционные**: Заставляет позиционные отслеживания, независимо от запросов системы.

Некоторые описанные выше понятия входной стоит на собственные. Тем не менее оставшиеся принципы могут применяться к одному из нескольких целевых объектов. Мы представили способа указать, какой использоваться вашей команды, которые должны быть применены к целевой.  В любом случае можно указать в пользовательском Интерфейсе или с помощью нажатия клавиатуры, которые объект targtet.  В некоторых случаях это также можно указать с помощью контроллера xbox напрямую.

В данной статье описываются полный набор типов входных данных и режимы ввода, которые доступны в эмуляторе HoloLens и симулятор Windows смешанной реальности. Для повседневного использования рекомендуется следующий набор элементов управления:

|  Операция |  Клавиатуры и мыши |  Контроллер | 
|----------|----------|----------|
|  Текст X |  A И D |  Левый джойстик влево / вправо | 
|  Текст Y |  Страницу вверх / страницу вниз |  DPad вверх / вниз | 
|  Текст Z |  W / S |  Левый джойстик вверх / вниз | 
|  Нутации текст |  Перетащите указатель мыши влево / вправо |  Правом джойстик влево / вправо | 
|  Поворот head |  H + перетаскивание мыши влево / вправо |  H (на клавиатуре) + вправо джойстик влево / вправо | 
|  Шаг head |  Перетащите указатель мыши вверх / вниз |  Джойстик вправо, вверх / вниз |

|  Рука X |  Alt + перетаскивание мыши влево / вправо |  Повышение + вправо джойстик влево / вправо |

|  Отслеживание |  T |  Кнопка х |
