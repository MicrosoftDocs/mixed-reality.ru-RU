# <a name="gestures"></a>Жесты

Жестами руками позволяют пользователям принимать меры в смешанной реальности. Взаимодействия основана на **помощи** к целевому объекту и **жест** или **голоса** действия целевых любой элемент. Жестами руками не предоставляют точное расположение в пространстве, но простота размещение на HoloLens и непосредственно взаимодействовать с содержимым позволяет приступить к работе без любые другие устройства.

## <a name="the-two-core-gestures-of-hololens"></a>Двумя основными жесты для HoloLens

В настоящее время HoloLens распознает два основных составляющих жестов - **жест касания** и **Блума**. Эти два основных взаимодействия находятся на низшем уровне пространственных входных данных, разработчик может получить доступ. Они формируют основу для разнообразных возможные действия пользователя.

### <a name="air-tap"></a>Жест касания

Жест касания находится жест касания с эксплуатационных вертикально, как щелчок мыши или выберите. Этот объект используется в большинстве возможности HoloLens для эквивалентных «щелкните» элемент пользовательского интерфейса после указавших его с [помощи](gaze.md). Это универсальный действие, которое Узнайте один раз и затем применить для всех приложений. Другие способы выполнения select являются нажатием одной кнопки [HoloLens Clicker](hardware-accessories.md#hololens-clicker) или голосом голосовые команды «Выбрать».

![Состояние готовности для жестами руками в HoloLens](images/image9.png)<br>
*Состояние готовности для жест касания в HoloLens.*

Касание является дискретных жест. Выбор либо завершения или не является, действие является или не будет выполнено в течение интерфейс. Это возможно, хотя это не рекомендуется без какой-либо определенной цели, для создания других дискретных жестов из комбинации основные компоненты (например жест двойного касания означает нечто иное, чем одно касание).

### <a name="bloom"></a>Раскрытия

Раскрытия является «home» жестов и зарезервирован для то отдельно. Это действие специальных системы, которое используется, чтобы вернуться в меню "Пуск". Это эквивалентно Windows клавиши на клавиатуре или кнопку Xbox на контроллере Xbox. Пользователь может использовать либо вручную.

![Жест раскрытия в HoloLens](images/image10-640px.png)

Чтобы сделать жест bloom на HoloLens, удерживать свои силы, palm, с помощью удобных друг с другом. Затем откройте свои силы. Обратите внимание, что вы также всегда можете вернуться на начальном произнеся «Привет, Кортана, Go Home». Приложения не может реагировать на них специально домашней действие, так как эти параметры обрабатываются системой.

### <a name="gesture-recognizers"></a>Средства распознавания жестов

Одним из преимуществ использования распознавания жестов — это, можно настроить распознаватель жестов для жестов, принимаемых текущей целевой голограмма. Платформа будет выполнять только неоднозначности, необходимые для различения этих определенного поддерживаемых жестов. Таким образом, голограмма, который поддерживает только жест касания может принимать любого периода времени между press и выпуска, во время голограмма что поддерживает коснитесь и удерживайте можно повысить уровень коснитесь, чтобы удержание после пороговое значение времени удержания.

## <a name="hand-recognition"></a>Распознавание вручную

HoloLens распознает жестами руками, отслеживая положение одного или обоих этих руки, видимых на устройстве. HoloLens видит руки, если они находятся в либо **Готово состояние** (лицевой стороны, с которыми сталкиваются с указательный палец) или **нажат** (лицевой стороны, с которыми сталкиваются с вытянутым указательным пальцем). Если в других создает руки, HoloLens будет игнорировать их.

Для каждой стрелки обнаруживает, HoloLens, можно получить доступ к его положение (не ориентация) и его состоянии нажатия. Как Рука приближается к краю фрейма жестов, вы также предоставляются с направление вектор, который может отображать для пользователя, чтобы они могли переместить их вручную, чтобы получить его обратно, где его можно просмотреть HoloLens.

## <a name="gesture-frame"></a>Жест кадра

Для жестов, HoloLens Рука должны находиться в жест фрейм – это, в диапазоне, камеры определением жестов на видно, соответствующим образом (очень примерно нос для уши, а также между плечи). Пользователи должны пройти обучение этой области распознавания как для успешного завершения действия, так и для собственных комфорта (многие пользователи будут изначально предполагается, что жест кадр должен быть в пределах их представления через HoloLens и держатся их оружию uncomfortably для взаимодействия). При использовании HoloLens Clicker, рук не обязательно должны находиться в пределах кадра жест.

В случае непрерывного жестов в частности, есть риск, некоторых пользователей перемещения опускают руки за пределами кадра жестов в середине жестов (во время перемещения некоторых holographic объекта, к примеру) и потере их желаемому результату.

Существует три вещи, которые следует учитывать:
* Обучение пользователей на жест рамки существование и приблизительное границы (это при изучении используется во время установки HoloLens).
* Уведомление пользователей о их жесты вскоре истекает и критические границы рамки жестов в приложении, гибко, потеряны жест приведет к нежелательным результаты. Исследования показали, ключевых качеств системы уведомления и HoloLens оболочка предоставляет хороший пример такого рода уведомлений (визуальный элемент, для центра курсора, указывающее направление, в какие границ пересечения выполняется).
* Следует свести к минимуму последствия нарушения работы границы рамки жест. Как правило это означает, что результат жеста должно быть остановлено на границе, однако не в обратном порядке. Например, если пользователь перемещается какой-либо объект holographic двух зала, перемещения следует остановить при нарушении кадра жестов, но **не** возвращаться к начальной точке. Пользователь может возникать некоторые разочарование затем может быстрее определить границы слоя, но не нужно перезапускать при каждом их полного необходимые действия.
