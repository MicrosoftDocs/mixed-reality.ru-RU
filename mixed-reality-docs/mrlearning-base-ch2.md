# <a name="mr-learning-base-module---user-interface-hand-tracking-and-mixed-reality-toolkit-configuration"></a>Обучения основной модуль - интерфейс пользователя MR вручную отслеживания и смешанные конфигурации набора средств реальность

В предыдущем уроке вы узнали некоторые возможности, которые может предложить, запустив приложение для HoloLens 2 MRTK. На следующем занятии вы узнаете, как создать и упорядочить кнопки, а также панели текста пользовательского интерфейса и использовать взаимодействие по умолчанию (касания) для взаимодействия с каждой кнопки. Также будет изучено добавление простых действий и эффекты, такие как изменение размера, звука и цвет объектов. Этот модуль будут представлены основные понятия о том, как изменять профили MRTK, начиная с отключение визуализации пространственных сетки. 

## <a name="objectives"></a>Цели

* Настройка и настроить профили смешанной реальности Toolkit
* Взаимодействие с голограммы, с помощью кнопок и элементов пользовательского интерфейса
* Базовой ввода вручную отслеживания и взаимодействия

## <a name="instructions"></a>Инструкция

### <a name="how-to-configure-the-mixed-reality-toolkit-profiles-change-spatial-awareness-display-option"></a>Настройка профилей Toolkit смешанной реальности (параметр отображения пространственных Awareness изменения)
В этом разделе вы узнаете, как настроить и настроить профили смешанной реальности Toolkit по умолчанию, изменив настройки отображения пространственных Кодекс сетки. Вы можете выполнить эти же принципы, для настройки параметров и значений в профилях MRTK.

1. Выберите набор средств Mixed реальности (MRTK) из иерархии «BaseScene». На панели Инспектора найдите «Смешанной реальности Toolkit скрипт» и выберите «active профиль», как показано на рисунке ниже. Дважды щелкните, чтобы открыть его.

![MR213_BuildSettings](images/mrlearning-base-ch2-1step1im.PNG)

>Примечание. По умолчанию профили MRTK недоступны для редактирования. Это шаблоны профиль по умолчанию, из которых можно скопировать и адаптировать. Существуют несколько уровней, настройки и профили, поэтому он является стандартной практикой, чтобы скопировать и адаптировать несколько профилей при настройке один или несколько параметров.
>
>Чтобы узнать больше о MRTK профилей и их архитектуры, см. на [MRTK документации](<https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/Architecture/SpatialAwareness/SpatialAwarenessSystemArchitecture.html>).

2. Создайте копию профиля по умолчанию для его настройки. Чтобы скопировать профиль по умолчанию, нажмите кнопку «Копировать & «Настройка» кнопку (см. рисунок). Это создает копию профиля MRTK. С помощью собственной копии MRTK профиля теперь у вас есть возможность изменить параметры в этом профиле. Также необходимо будет повторить процедуру копирования или настроить для любой дополнительные профили, вложенные в этот профиль, как описано в последующих шагах.

![MR213_BuildSettings](images/mrlearning-base-ch2-1step2im.PNG)

3. Отключите видимость пространственных осведомленности в области сетки. Для этого найдите «Параметры системы пространственных Awareness», как показано на рисунке. Нажмите кнопку «клонировать» справа от «Пространственных Awareness системный профиль» заменить копию настраиваемый профиль по умолчанию. Если появится всплывающее окно, нажмите кнопку «Клонировать», как показано на приведенном ниже рисунке второй.

![MR213_BuildSettings](images/mrlearning-base-ch2-1step3im.PNG)

![MR213_BuildSettings](images/mrlearning-base-ch2-1step3bim.jpg)

4. Создание пользовательских копий по умолчанию смешанной реальности пространственных Mesh наблюдателя. Щелкните стрелку вниз рядом с «Windows смешанной реальности пространственных Mesh наблюдатель» для просмотра дополнительных параметров. В эти параметры вы увидите, что «По умолчанию смешанной реальности пространственных Mesh наблюдатель» которого отображается серым (невозможно изменить). Этот профиль по умолчанию необходимо заменить копию настраиваемых, поэтому его можно изменить. Нажмите кнопку справа (отмечаются зеленой стрелкой), чтобы клонировать копию.

![MR213_BuildSettings](images/mrlearning-base-ch2-1step4im.PNG)

5. Теперь можно отрегулировать настройки для параметра отображения сказать «перекрытия.» Это делает пространственных сетки невидимым, но по-прежнему скрыть игр объекты позади пространственных сетки (это называется перекрытия.)

![MR213_BuildSettings](images/mrlearning-base-ch2-1step5im.PNG)

>Примечание. Хотя пространственное сопоставление сети не отображается, по-прежнему присутствует, и вы можете взаимодействовать с ним. Любой голограммы за пространственное сопоставление сети (например голограмма за видимой стене), не будут видны из-за параметра перекрытия.

Поздравляем! Вы только что узнали, как изменить параметр в профиле MRTK. Как вы видите, чтобы изменить параметры MRTK, которые необходимы для создания копий из профилей по умолчанию, таким образом, чтобы их можно изменять. Вы всегда сможете профили по умолчанию (которые будут недоступны для редактирования) чтобы вернуться к Если вы хотите создать профиль с использованием новых параметров, или обращаться к профили по умолчанию. Существует множество параметров, которые можно настраивать. Параметры профиля MRTK полную ссылку см. в документации MRTK: https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/GettingStartedWithTheMRTK.html

### <a name="hand-tracking-gestures-and-interactable-buttons"></a>Отслеживание жестами руками и элементом кнопки
В этом разделе вы узнаете способы использования отслеживания к кнопке элементом вручную.

1. Выберите «Активы» в папке проектов.

2. Введите в строке поиска «pressablebutton.»

![MR213_BuildSettings](images/mrlearning-base-ch2-2step1im.PNG)

3. Перетащите готового блока (представленный прямоугольником синего) с именем «PressableButton» в иерархии. 

   > Примечание. Если вы получите сообщение «импорт TMP Essentials» импортируйте его в данный момент. Если TMP Essentials еще не было частью проекта, может потребоваться повторить этот шаг после импорта TMP Essentials, в противном случае текст кнопки может не отображаться.

![MR213_BuildSettings](images/mrlearning-base-ch2-2step3im.PNG)

4. Дважды щелкните объект игр «PressableButton» (который теперь должен находиться в иерархии BaseScene) на его просмотр в сцене, как показано на рисунке ниже (графики кнопки может выглядеть иначе, чем на рисунке ниже). 

![MR213_BuildSettings](images/mrlearning-base-ch2-2step4im.PNG)

5. На панели Инспектора нажмите кнопку «Добавить событие», чтобы присвоить кнопке реагировать при передаче события. В тот вариант, который отображается выберите стрелку раскрывающегося меню. Выберите «InteractableOnPressReciever.» Это позволяет кнопке реагировать на событие нажатия, когда отслеживаемые руки нажимает кнопку.

![MR213_BuildSettings](images/mrlearning-base-ch2-2step5im.PNG)

6. Добавьте в сцену куб. Щелкните правой кнопкой мыши в области иерархии, выберите трехмерный объект, а затем щелкните «куб». Теперь куб должен находиться в вашего экрана. Он будет отображаются очень большими, поэтому настроить координаты (хотя «куб» по-прежнему выбран в области иерархии) для уменьшения размера. Задайте масштаб значения x = 0,1, y = 0,1 и z = 0,1. Быть в том изменить расположение куба в сцене разместить его рядом с кнопкой pressable, но не перекрывающихся с кнопкой. В приведенном ниже рисунке куба расположена x = 0, y = 0.2 и z = 1. 

   > Примечание. В общем случае 1 единицу в Unity примерно равно 1 метра в реальном мире. Существуют исключения, например, когда объекты являются дочерними масштабированное объектов.
   
   ![MR213_BuildSettings](images/mrlearning-base-ch2-1step6ima.PNG)

![MR213_BuildSettings](images/mrlearning-base-ch2-2step6imb.PNG)

7. На этом шаге вы настроите куба для изменения цвета при нажатии кнопки. Выберите PressableButton в меню «BaseScene». На панели Инспектора под полем «Выберите тип события», нажмите кнопку «+». Перетащите «куб» в меню «BaseScene» в поле «Только для среды выполнения», как показано на рисунке ниже

![MR213_BuildSettings](images/mrlearning-base-ch2-2step7ima.PNG)

Щелкните раскрывающийся список с текстом «Не функция». Выберите «MeshRenderer», а затем выберите «Материала материал». Это позволит вам изменить материала, при нажатии кнопки. 

![MR213_BuildSettings](images/mrlearning-base-ch2-2step7imb.PNG)

Перейдите в панель «проект» и поиска для материала, который вы хотите изменить его. Вы собираетесь использовать материал «MRTK_Standard_Cyan» в этом примере, найти, введя в «голубой» в строке поиска на вкладке проекта (MRTK включает множество материалов и цветов.) Перетащите материала в поле под «MeshRenderer.material.» MRTK материалы можно найти в ресурсах > MixedRealityToolkit.SDK > StandardAssets > материалов.

![MR213_BuildSettings](images/mrlearning-base-ch2-2step7imbb.PNG)

![MR213_BuildSettings](images/mrlearning-base-ch2-1step7imc.PNG)

Это событие теперь настроен таким образом, чтобы при нажатии кнопки, куб будет меняться на основе материалов, которые вы указали. В этом примере куба изменится на голубого цвета.

8. Далее нужно настроить действие выпуска, чтобы после ее выхода, кнопку вернется к цвета по умолчанию. Повторите шаг 7 (выше) но на этот раз с событием «OnRelease» вместо «OnPress» события, как показано на рисунке ниже.

9.  В панели «проект» поиск материал для куба по умолчанию после кнопки выпуска (в нашем случае мы выбрали MRTK_Standard_LightGray.) Перетащите материала в поле под полем «MeshRenderer.material».

![MR213_BuildSettings](images/mrlearning-base-ch2-2step8im.PNG)

Теперь при нажатии кнопки, его следует изменить на новый цвет (например cyan), а при отпускании кнопки его следует меняет цвет по умолчанию, который вы указали (например, светло-серый.) Нажмите кнопку «Воспроизведение» в верхней части экрана, чтобы опробовать его в редакторе или развернуть в вашем HoloLens 2 для тестирования! Дополнительные сведения о в редакторе моделирования, включая чтение моделирование вручную [странице документации по MRTK моделирования](<https://microsoft.github.io/MixedRealityToolkit-Unity/Documentation/InputSimulation/InputSimulationService.html>). 

### <a name="creating-a-panel-of-buttons-using-mrtks-grid-object-collection"></a>Создание панели кнопок, с помощью коллекции объектов MRTK элемента сетки

В этом разделе вы узнаете, как для автоматического выравнивания несколько кнопок в ясном пользовательский интерфейс, с помощью средства GridObjectCollection MRTK.

1. Дублирование кнопки из предыдущего раздела, пока не получится 5 кнопок. Существует несколько способов это сделать:
- Щелкните правой кнопкой мыши кнопку «Копировать», а затем перейдите вниз под кнопкой и снова щелкните правой кнопкой мыши и щелкните «Вставить». 
- Щелкните правой кнопкой мыши кнопку и нажмите кнопку «дублирование». 
- Команда клавиатуры, щелкнув куб и нажатия клавиши «ctrl D» на клавиатуре.

Повторите эти действия, пока не получится 5 общее кнопок (см. в разделе 5 красные стрелки на рисунке ниже).

![3Step1im Ch2 Mrlearning Base](images/mrlearning-base-ch2-3step1im.PNG)

2. Группы кнопок в пустой родительский объект игр. Чтобы у кнопок в коллекции сетки, необходимо сгруппировать к кнопкам в общий родительский объект. Щелкните правой кнопкой мыши hiearachy и нажмите кнопку «Создать пустой». Это создает новый пустой объект игр для можно поместить все кнопки. Он будет отображаться как «gameObject.» Щелкните правой кнопкой мыши и переименуйте его в «ButtonCollection.»

![3Step2im Ch2 Mrlearning Base](images/mrlearning-base-ch2-3step2im.PNG)

3. Переместите все кнопки в новой коллекции. Это сделать, выбрав все пять объектов button в вашей heirarch и перетащите их все в разделе «ButtonCollection» игровому объекту, как показано на рисунке ниже. Совет: выберите несколько элементов, удерживая клавишу ctrl при выборе элементов.

![3Step3imb Ch2 Mrlearning Base](images/mrlearning-base-ch2-3step3imb.PNG)

4. Добавьте компонент «Коллекции объектов сетки» MRTK элемента коллекции кнопок панели.  Чтобы сделать это, выберите родительский объект «ButtonCollection» и на панели инспектора, нажмите кнопку «Добавить компонент». Выполните поиск «Коллекции объектов сетки» в строке поиска и выберите его, когда он появится в списке.

![3Step4im Ch2 Mrlearning Base](images/mrlearning-base-ch2-3step4im.PNG)

Компонент сетки коллекции объектов позволяет упорядочить кнопки или любой набор объектов в ясном строки, столбца или сетки. Это один из блоков, предоставляемые MRTK, обеспечивает простой и быстрый способ создания привлекательных пользовательских интерфейсов.

5. Настройка сбора объект сетки. Чтобы обеспечить все поверхности кнопки пользователю, выберите «ориентация тип», а затем выберите «сталкиваются родительского вперед» (как показано на рисунке.) Затем измените размер ячейки, чтобы задать интервал между своим кнопкам. Начните с 0,12 единиц по 0,12 единицы для ячейки ширину и высоту ячейки, как показано на рисунке ниже. Может потребоваться изменить эти значения, в зависимости от выбранного активы объекта или кнопки. Убедитесь, что «Строки» имеет значение 1. Нажмите кнопку «Обновить коллекцию» и сцены должен выглядеть, как показано на рисунке ниже. 


![3Step5im Ch2 Mrlearning Base](images/mrlearning-base-ch2-3step5im.PNG)

>Примечание. В зависимости от ориентации дочерних объектов или родительский объект, вы скорее всего, потребуется изменить ориентацию, параметр по-разному, в будущих проектах. Ширина и высота ячейки поля может также должны быть определены по-разному, в зависимости от размера объектов в коллекции.

### <a name="adding-text-into-your-scene"></a>Добавление текста в сценах

В этом разделе вы узнаете, как добавлять и редактировать текст, который своим опытом смешанной реальности. Если это еще не сделано, убедитесь, включена в Unity, следуя инструкциям TextMeshPro [здесь](https://docs.unity3d.com/Packages/com.unity.textmeshpro@2.0/manual/index.html#installation).

1. Выберите родительский объект «ButtonCollection» и щелкните правой кнопкой мыши коллекции. Разверните узел «трехмерный объект» в раскрывающемся меню, а затем выберите «TextMeshPro – Text». Вы должны увидеть TextMeshPro объект, находящийся в коллекции кнопок панели, как показано на рисунке ниже.

![Занятие 2 Chapter4 Step1a](images/Lesson2_Chapter4_Step1a.JPG)
![Step1b Chapter4 занятие 2](images/Lesson2_Chapter4_Step1b.JPG)

2. В текстовом поле TextMeshPro компонента (находится на панели инспектора, как показано ниже) введите «Коллекции текста кнопки». Текст будет отображаться в сцене, но она будет скрыта за кнопок и/или неправильный размер.

![Шаг 2 Chapter4 занятие 2](images/Lesson2_Chapter4_Step2.JPG)

3. Чтобы повысить размер текста и размещения для удобства чтения, измените поле «размер шрифта» в компоненте TextMeshPro, чтобы изменить размер шрифта. Также необходимо будет настроить «Rect преобразованием» положение и размер, как показано на рисунке ниже. См. в разделе ниже образы для значения, используемые для нашей конфигурации текст и вы можете использовать эти значения в качестве отправной точки, из которого необходимо увеличить размер и размещение текстовое поле.

![Этап 3 Chapter4 занятие 2](images/Lesson2_Chapter4_Step3.JPG)
![шаг 4 Chapter4 занятие 2](images/Lesson2_Chapter4_Step4.JPG)

5. Чтобы изменить текстовые значения в объектах кнопку, щелкните стрелку рядом с любой кнопки, чтобы развернуть его и перейдите к объекту «SeeItSayItLabel», а затем перейдите к «TextMeshPro», где можно изменить текст к своим кнопкам, как описано ранее.

![Шаг 5 Chapter4 занятие 2](images/Lesson2_Chapter4_Step5.JPG)
