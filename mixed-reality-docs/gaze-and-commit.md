### <a name="navigation"></a><span data-ttu-id="72a41-243">Навигация</span><span class="sxs-lookup"><span data-stu-id="72a41-243">Navigation</span></span>
<span data-ttu-id="72a41-244">Жесты навигации действуют как виртуального джойстика и может использоваться для перемещения мини-приложения пользовательского интерфейса, таких как радиального меню.</span><span class="sxs-lookup"><span data-stu-id="72a41-244">Navigation gestures operate like a virtual joystick, and can be used to navigate UI widgets, such as radial menus.</span></span> <span data-ttu-id="72a41-245">Коснитесь и удерживайте, чтобы запустить данный жест, а затем переместите свои силы в нормализованный трехмерный куб, сосредоточенным начальной нажмите клавишу.</span><span class="sxs-lookup"><span data-stu-id="72a41-245">You tap and hold to start the gesture and then move your hand within a normalized 3D cube, centered around the initial press.</span></span> <span data-ttu-id="72a41-246">Можно переместить свои силы на оси X, Y или Z из значение -1, 1, причем 0 представляет начальную точку.</span><span class="sxs-lookup"><span data-stu-id="72a41-246">You can move your hand along the X, Y or Z axis from a value of -1 to 1, with 0 being the starting point.</span></span>

<span data-ttu-id="72a41-248">Переход с помощью rails обозначает возможность распознавать перемещений в определенной оси, до достижения определенного порогового значения на этой оси.</span><span class="sxs-lookup"><span data-stu-id="72a41-248">Navigation with rails refers to the ability of recognizing movements in certain axis until certain threshold is reached on that axis.</span></span> <span data-ttu-id="72a41-249">Это полезно только, если перемещение в более чем одну ось включена в приложении разработчиком, например если приложение распознает жесты навигации по X, ось Y, а также указаны X ось rails.</span><span class="sxs-lookup"><span data-stu-id="72a41-249">This is only useful, when movement in more than one axis is enabled in an application by the developer, e.g. if an application is configured to recognize navigation gestures across X, Y axis but also specified X axis with rails.</span></span> <span data-ttu-id="72a41-250">В этом случае система распознает наличии перемещений по оси X до тех пор, пока они остаются в пределах мнимой шин (руководство по) на оси X, при наличии перемещение также происходит по оси Y.</span><span class="sxs-lookup"><span data-stu-id="72a41-250">In this case system will recognize hand movements across X axis as long as they remain within an imaginary rails (guide) on X axis, if hand movement also occurs Y axis.</span></span>

<span data-ttu-id="72a41-251">В двухмерных приложений пользователи могут использовать жесты для навигации по вертикали для прокрутки, масштаб или перетащите его в приложении.</span><span class="sxs-lookup"><span data-stu-id="72a41-251">Within 2D apps, users can use vertical navigation gestures to scroll, zoom, or drag inside the app.</span></span> <span data-ttu-id="72a41-252">Это внедряет штрихи виртуального палец в приложение для имитации сенсорные жесты того же типа.</span><span class="sxs-lookup"><span data-stu-id="72a41-252">This injects virtual finger touches to the app to simulate touch gestures of the same type.</span></span> <span data-ttu-id="72a41-253">Пользователи могут выбрать, какие из этих действий выполняются с помощью переключения между инструментами на строке в верхней части приложения, нажав кнопку или о том, «< прокрутки или перетащите/> масштаба».</span><span class="sxs-lookup"><span data-stu-id="72a41-253">Users can select which of these actions take place by toggling between the tools on the bar above the app, either by selecting the button or saying '<Scroll/Drag/Zoom> Tool'.</span></span>

## <a name="gesture-recognizers"></a><span data-ttu-id="72a41-255">Средства распознавания жестов</span><span class="sxs-lookup"><span data-stu-id="72a41-255">Gesture recognizers</span></span>

<span data-ttu-id="72a41-256">Одним из преимуществ использования распознавания жестов — это, можно настроить распознаватель жестов для жестов, принимаемых текущей целевой голограмма.</span><span class="sxs-lookup"><span data-stu-id="72a41-256">One benefit of using gesture recognition is that you can configure a gesture recognizer just for the gestures the currently targeted hologram can accept.</span></span> <span data-ttu-id="72a41-257">Платформа будет выполнять только неоднозначности, необходимые для различения этих определенного поддерживаемых жестов.</span><span class="sxs-lookup"><span data-stu-id="72a41-257">The platform will do only the disambiguation necessary to distinguish those particular supported gestures.</span></span> <span data-ttu-id="72a41-258">Таким образом, голограмма, который поддерживает только жест касания может принимать любого периода времени между press и выпуска, во время голограмма что поддерживает коснитесь и удерживайте можно повысить уровень коснитесь, чтобы удержание после пороговое значение времени удержания.</span><span class="sxs-lookup"><span data-stu-id="72a41-258">That way, a hologram that just supports air tap can accept any length of time between press and release, while a hologram that supports both tap and hold can promote the tap to a hold after the hold time threshold.</span></span>

## <a name="hand-recognition"></a><span data-ttu-id="72a41-259">Распознавание вручную</span><span class="sxs-lookup"><span data-stu-id="72a41-259">Hand recognition</span></span>
<span data-ttu-id="72a41-260">HoloLens распознает жестами руками, отслеживая положение одного или обоих этих руки, видимых на устройстве.</span><span class="sxs-lookup"><span data-stu-id="72a41-260">HoloLens recognizes hand gestures by tracking the position of either or both hands that are visible to the device.</span></span> <span data-ttu-id="72a41-261">HoloLens видит руки, если они находятся в состоянии готовности (лицевой стороны, с которыми сталкиваются с указательный палец) или нажаты (лицевой стороны, с которыми сталкиваются с вытянутым указательным пальцем).</span><span class="sxs-lookup"><span data-stu-id="72a41-261">HoloLens sees hands when they are in either the ready state (back of the hand facing you with index finger up) or the pressed state (back of the hand facing you with the index finger down).</span></span> <span data-ttu-id="72a41-262">Если в других создает руки, HoloLens будет игнорировать их.</span><span class="sxs-lookup"><span data-stu-id="72a41-262">When hands are in other poses, the HoloLens will ignore them.</span></span>
<span data-ttu-id="72a41-263">Для каждой стрелки обнаруживает, HoloLens, можно получить доступ к его положение (не ориентация) и его состоянии нажатия.</span><span class="sxs-lookup"><span data-stu-id="72a41-263">For each hand that HoloLens detects, you can access its position (without orientation) and its pressed state.</span></span> <span data-ttu-id="72a41-264">Как Рука приближается к краю фрейма жестов, вы также предоставляются с направление вектор, который может отображать для пользователя, чтобы они могли переместить их вручную, чтобы получить его обратно, где его можно просмотреть HoloLens.</span><span class="sxs-lookup"><span data-stu-id="72a41-264">As the hand nears the edge of the gesture frame, you're also provided with a direction vector, which you can show to the user so they know how to move their hand to get it back where HoloLens can see it.</span></span>

## <a name="gesture-frame"></a><span data-ttu-id="72a41-265">Жест кадра</span><span class="sxs-lookup"><span data-stu-id="72a41-265">Gesture frame</span></span>
<span data-ttu-id="72a41-266">Для жестов, HoloLens Рука должны находиться в жест фрейм – это, в диапазоне, камеры определением жестов на видно, соответствующим образом (очень примерно нос для уши, а также между плечи).</span><span class="sxs-lookup"><span data-stu-id="72a41-266">For gestures on HoloLens, the hand must be within a “gesture frame”, in a range that the gesture-sensing cameras can see appropriately (very roughly from nose to waist, and between the shoulders).</span></span> <span data-ttu-id="72a41-267">Пользователи должны пройти обучение этой области распознавания как для успешного завершения действия, так и для собственных комфорта (многие пользователи будут изначально предполагается, что жест кадр должен быть в пределах их представления через HoloLens и держатся их оружию uncomfortably для взаимодействия).</span><span class="sxs-lookup"><span data-stu-id="72a41-267">Users need to be trained on this area of recognition both for success of action and for their own comfort (many users will initially assume that the gesture frame must be within their view through HoloLens, and hold their arms up uncomfortably in order to interact).</span></span> <span data-ttu-id="72a41-268">При использовании HoloLens Clicker, рук не обязательно должны находиться в пределах кадра жест.</span><span class="sxs-lookup"><span data-stu-id="72a41-268">When using the HoloLens Clicker, your hands do not need to be within the gesture frame.</span></span>

<span data-ttu-id="72a41-269">В случае непрерывного жестов в частности, есть риск, некоторых пользователей перемещения опускают руки за пределами кадра жестов в середине жестов (во время перемещения некоторых holographic объекта, к примеру) и потере их желаемому результату.</span><span class="sxs-lookup"><span data-stu-id="72a41-269">In the case of continuous gestures in particular, there is some risk of users moving their hands outside of the gesture frame while in mid-gesture (while moving some holographic object, for example), and losing their intended outcome.</span></span>

<span data-ttu-id="72a41-270">Существует три вещи, которые следует учитывать:</span><span class="sxs-lookup"><span data-stu-id="72a41-270">There are three things that you should consider:</span></span>

- <span data-ttu-id="72a41-271">Обучение пользователей на жест рамки существование и приблизительное границы (это при изучении используется во время установки HoloLens).</span><span class="sxs-lookup"><span data-stu-id="72a41-271">User education on the gesture frame's existence and approximate boundaries (this is taught during HoloLens setup).</span></span>

- <span data-ttu-id="72a41-272">Уведомление пользователей о их жесты вскоре истекает и критические границы рамки жестов в приложении, гибко, потеряны жест приведет к нежелательным результаты.</span><span class="sxs-lookup"><span data-stu-id="72a41-272">Notifying users when their gestures are nearing/breaking the gesture frame boundaries within an application, to the degree that a lost gesture will lead to undesired outcomes.</span></span> <span data-ttu-id="72a41-273">Исследования показали, ключевых качеств системы уведомления и HoloLens оболочка предоставляет хороший пример такого рода уведомлений (визуальный элемент, для центра курсора, указывающее направление, в какие границ пересечения выполняется).</span><span class="sxs-lookup"><span data-stu-id="72a41-273">Research has shown the key qualities of such a notification system, and the HoloLens shell provides a good example of this type of notification (visual, on the central cursor, indicating the direction in which boundary crossing is taking place).</span></span>

- <span data-ttu-id="72a41-274">Следует свести к минимуму последствия нарушения работы границы рамки жест.</span><span class="sxs-lookup"><span data-stu-id="72a41-274">Consequences of breaking the gesture frame boundaries should be minimized.</span></span> <span data-ttu-id="72a41-275">Как правило это означает, что результат жеста должно быть остановлено на границе, однако не в обратном порядке.</span><span class="sxs-lookup"><span data-stu-id="72a41-275">In general, this means that the outcome of a gesture should be stopped at the boundary, but not reversed.</span></span> <span data-ttu-id="72a41-276">Например если пользователь перемещается какой-либо объект holographic двух зала, перемещение останавливаться при фрейма жестов, нарушена, но не возвращаться к начальной точке.</span><span class="sxs-lookup"><span data-stu-id="72a41-276">For example, if a user is moving some holographic object across a room, movement should stop when the gesture frame is breached, but not be returned to the starting point.</span></span> <span data-ttu-id="72a41-277">Пользователь может возникать некоторые разочарование затем может быстрее определить границы слоя, но не нужно перезапускать при каждом их полного необходимые действия.</span><span class="sxs-lookup"><span data-stu-id="72a41-277">The user may experience some frustration then, but may more quickly understand the boundaries, and not have to restart their full intended actions each time.</span></span>
