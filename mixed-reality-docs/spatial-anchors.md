# <a name="spatial-anchors"></a>Пространственные привязки

Пространственные привязки представляет важный момент в мире, система должна хранить список со временем. Каждая привязка имеет [системы координат](coordinate-systems.md) корректировать при необходимости относительно других привязок, или поместите границы отсчета, чтобы гарантировать, что привязанный голограммы остаются точно в.  Подготовка к просмотру голограмма в системе координат привязки дает наиболее точную позиционирование для этого голограмма в любой момент времени. Это достигается за счет код небольшие поправки со временем в положение голограмма, как система постоянно перемещается обратно в расположение относительно реального мира.

Кроме того, можно также сохранять и совместно использовать Пространственные привязки между сеансами приложения и с любого устройства:
* Сохранив локальные пространственных привязки на диск и при загрузке их позже, приложение можно рассуждать о расположении в реальном мире для нескольких сеансов приложения на одно устройство HoloLens.
* С помощью <a href="https://docs.microsoft.com/azure/spatial-anchors/overview" target="_blank">привязки пространственных Azure</a> создать привязку в облаке, приложения могут совместно использовать Пространственные привязки между несколькими HoloLens, iOS и устройств Android. Назначив каждому устройству отрисовки голограмма, с помощью одной пространственных привязкой, все пользователи будут видеть голограмма, которые отображаются в том же месте, в реальном мире.  Это позволяет выполнять публикацию в режиме реального времени.
* Можно также использовать <a href="https://docs.microsoft.com/azure/spatial-anchors/overview" target="_blank">привязки пространственных Azure</a> для асинхронной голограмма сохраняемости на устройствах Android, iOS и HoloLens.  По электронной почте на привязку пространственных надежные облачные, несколько устройств можно наблюдать за же материализованных голограмма со временем, даже если эти устройства не существуют друг с другом в то же время.

Для фиксированный масштаб или масштаб комнаты работы в связанном применяется настольных систем, которые будут оставаться в пределах диаметр 5-индикатор, обычно можно использовать [этап подразумевает](coordinate-systems.md#stage-frame-of-reference) вместо пространственных привязки, предоставляя один система координат, в котором будет подготовлен все содержимое. Тем не менее считал приложения позволяют пользователям, которые перемещаются за пределы 5 м на HoloLens, возможно операционной на протяжении всей этаж в здании, вам потребуется пространственных привязки для сохранения содержимого стабильная версия.

Во время привязки пространственных отлично подходят для голограммы, которые следует зафиксировать в мире, поместив привязки, ее нельзя переместить. Существуют альтернативы привязки, которые больше подходят для динамического голограммы, которые следует обозначить тегом, а также пользователя. Лучше изменить расположение динамического голограммы, с помощью стационарной системой отсчета координат (основа для Unity мировых координатах) или присоединенного отсчета координат.

## <a name="best-practices"></a>Рекомендации

Эти рекомендации пространственных привязки поможет отрисовки стабильной голограммы, точно отслеживающие реального мира.

### <a name="create-spatial-anchors-where-users-place-them"></a>Создавать Пространственные привязки, где пользователи помещают

В большинстве случаев, пользователи должны иметь тех, которые явно размещение пространственных привязки.

Например, на HoloLens, приложение может пересекаются пользователя [помощи](gaze.md) ray с [пространственное сопоставление](spatial-mapping.md) сетки, чтобы позволить пользователю указать место размещения голограмма. Когда пользователь касается для размещения этого голограмма, создайте привязку пространственных в точке пересечения, а затем поместите голограмма в начало системы координат эту привязку.

Локальный пространственных привязки просты и повышает производительность, однако создание и система будет консолидировать их внутренними данными, если несколько привязок могут совместно использовать их базовые данные датчика. Обычно следует создать новый локальный пространственных привязки для каждого голограмма, пользователь явным образом помещается, за исключением случаев, описанных ниже, такие как жесткая групп голограммы.

### <a name="always-render-anchored-holograms-within-3-meters-of-their-anchor"></a>Всегда выводить привязанного голограммы в трех ваттметров их привязки

Пространственные привязки стабилизировать систему координат практически источника привязки. При подготовке к просмотру голограммы более трех ваттметров от начала координат, эти голограммы могут испытывать заметно позиционные пропорционально расстоянию от начала координат, из-за эффекты бегунка arm. Это работает, если пользователь заменяет рядом с точкой привязки, так как она слишком далеко от пользователя, это означает, что angular ошибку отдаленных голограмма будет небольшим. Тем не менее если он идет, отдаленных голограмма, он будет большой в их представлении, что вполне очевидны эффекты бегунка arm от начала координат отдаленные привязки.

### <a name="group-holograms-that-should-form-a-rigid-cluster"></a>Голограммы группы, которые должны стать жесткой кластера

Несколько голограммы могут совместно использовать одной пространственных привязкой, если приложение ожидает, что эти голограммы для поддержания фиксированного связи друг с другом.

Например при анимации holographic Солнечная система в комнате, лучше привязать все объекты Солнечной системы на одном привязку в центре, перемещаемые плавно относительно друг друга. В этом случае это Солнечной системы в целом, к которому привязана, несмотря на то, что ее компонентов динамически перемещения привязки.

Ключа предостережение для поддержания стабильности голограмма — следовать приведенным выше правилом 3 метр.

### <a name="render-highly-dynamic-holograms-using-the-stationary-frame-of-reference-instead-of-a-local-spatial-anchor"></a>Визуализации высокодинамичных голограммы, вместо локального пространственных привязки стационарной системой отсчета координат

Если у вас есть высокодинамичных голограмма, такой как символ прогулке по комнате или с плавающей запятой пользовательский Интерфейс, который следует за вдоль стены рядом с пользователем, лучше пропустить локального пространственных привязки и отображения этих голограммы прямо в системе координат, предоставляемые [</C0настроекстационарнойсистемойотсчетакоординат<spanclass="notranslate">](coordinate-systems.md#stationary-frame-of-reference) (т. е. в Unity вы добиться этого, разместив голограммы прямо в мировых координатах без WorldAnchor).</span> Голограммы в стационарной системой отсчета координат могут возникнуть смещение, когда пользователь является далеко не голограмма, но это менее вероятно, будет заметно только для динамических голограммы: либо она постоянно движется вперед несмотря на это, либо ее движение постоянно отслеживает его рядом с пользователем , где будет минимальным смещение.

Один интересный случай из динамических голограммы — это объект, который выполняет анимацию из одной системы координат для привязанного к другому. Например возможно, двух контейнерах 10 метров, каждый на собственные Пространственные привязки, с одной castle срабатывание cannonball в других castle. В момент cannonball срабатывает можно отобразить его в соответствующее место в стационарной системой отсчета координат, таким образом, чтобы приурочить компании cannon в первом castle привязанного системе координат. Оно затем может следовать его траекторию в стационарной системой отсчета координат как его из Сиэттла отправится 10 метров по воздуху. Когда cannonball достигает других castle, вы можете перемещать его в второй castle привязанные систему координат, чтобы разрешить для физики вычисления с жесткой тел, castle.

Если вы используете высокодинамичных голограмма на устройствах, необходимо будет выбрать привязку пространственных некоторые в облаке в качестве родительского элемента, как стационарной системой отсчета кадры нельзя использовать совместно на устройствах.  Однако в этом случае следует убедиться, либо динамический голограмма или устройств, просмотрев его оставался в radius 3 индикатор точку привязки, чтобы убедиться, что она отображается стабильной на всех устройствах.

### <a name="avoid-creating-a-grid-of-spatial-anchors"></a>Не следует создавать сетку пространственных привязки

У вас может возникнуть желание drop обычный grid пространственных якорей, когда пользователь выходит, переход из привязки для привязки динамических объектов, как перемещения приложений. Тем не менее это подразумевает проведение большой управления для своего приложения, не преимущество данные глубокого датчика, сама система поддерживает внутренним образом. В этих случаях будет обычно добиться лучших результатов с меньшими трудозатратами, поместив вашей голограммы в стационарной системой отсчета координат, как описано в предыдущем разделе.

Предварительно положение набор пространственных привязки облака вокруг статический пробел, рассмотрите возможность размещения пространственных привязки в местах, из ключа голограммы, которыми пользователь столкнетесь на принцип выше, а не создавать произвольные сетку привязки.  Это гарантирует, что вы получите максимальный стабильности для этих ключей голограммы.

### <a name="release-local-spatial-anchors-you-no-longer-need"></a>Выпуск локального пространственных привязки, которые больше не требуется

При активном привязку локального Пространственные системы будет определять их приоритеты хранения вокруг данных датчиков, у которой эту привязку. Если вы больше не используется пространственных привязки, stop, доступ к его система координат. Таким образом, его базовые данные датчиков, удаляемый при необходимости.

Это особенно важно для локального привязки, которые они сохраняются в хранилище пространственных привязки. Данные датчика за эти привязки будут храниться вокруг окончательно чтобы разрешить приложению найти эту привязку в будущих сеансах приложения, что позволит уменьшить свободное место для отслеживания других привязок. Сохраняются только для локального привязки, которые необходимо найти в будущих сеансах и удалить их из хранилища, когда они больше не значимую для пользователя.

Для привязки пространственных облака хранилище можно масштабировать по мере сценарий требует.  Вы можете сохранить столько привязки облака, при необходимости, освобождая их, только в том случае, когда вы знаете, что пользователям не потребуется выполнять попытку найти голограммы во эту привязку.
