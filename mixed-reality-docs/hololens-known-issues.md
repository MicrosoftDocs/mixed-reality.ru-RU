# <a name="hololens-known-issues"></a>Известные проблемы HoloLens

Это текущий список известных проблем для HoloLens, влияющих на разработчиков. Сначала установите флажок, если вы видите необычное поведение. Этот список будет обновляться по мере обнаружения или сообщения новых проблем или в случае возникновения проблем в будущих обновлениях программного обеспечения HoloLens.

## <a name="unable-to-connect-and-deploy-to-hololens-through-visual-studio"></a>Не удается подключиться и выполнить развертывание в HoloLens через Visual Studio

>[!NOTE]
>Последнее обновление: 8/8 @ 5:11 вечера — Visual Studio выпустила VS 2019 версии 16,2, которая включает исправление этой проблемы. Рекомендуется выполнить обновление до этой последней версии, чтобы избежать этой ошибки.

Visual Studio выпустила VS 2019 версии 16,2, которая включает исправление этой проблемы. Рекомендуется выполнить обновление до этой последней версии, чтобы избежать этой ошибки.

Основная причина проблемы: Пользователи, которые использовали Visual Studio 2015 или ранние выпуски Visual Studio 2017 для развертывания и отладки приложений на их HoloLens, а затем использовали последние версии Visual Studio 2017 или Visual Studio 2019 с одним и тем же HoloLens, будут затронуты. Новые выпуски Visual Studio развертывают новую версию компонента, но файлы из старой версии остаются на устройстве, что приводит к сбою новой версии.  Это приводит к появлению следующего сообщения об ошибке: DEP0100: Убедитесь, что на целевом устройстве включен режим разработчика. Не удалось получить лицензию <ip> разработчика из-за ошибки 80004005.
 
**Инструкции по решению**. 

Наша группа сейчас работает над исправлением. В то же время можно выполнить следующие действия, чтобы устранить эту ошибку и помочь разблокировать развертывание и отладку.  
1. Откройте Visual Studio
2. Файл-> проект New->
3. Визуальное C# > Windows Desktop — > консольное приложение (.NET Framework)
4. Присвойте проекту имя (например, Хололенсдеплойментфикс) и убедитесь, что для платформы задано значение по крайней мере .NET Framework 4,5, а затем нажмите кнопку ОК.
5. Щелкните правой кнопкой мыши узел ссылки в обозреватель решений и добавьте следующие ссылки (щелкните в разделе "Обзор" и нажмите кнопку "Обзор". (кнопка)):
    ```
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Deploy.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Connectivity.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\SirepInterop.dll
    ```
    >[!NOTE]
    >Если вы не установили 10.0.18362.0, используйте самую последнюю версию.
 
6. Щелкните правой кнопкой мыши проект в обозреватель решений и выберите Добавить-> существующий элемент.
 
7. Перейдите к папке C:\Program Files (x86) \Windows Kits\10\bin\10.0.18362.0\x86 и измените фильтр на "все файлы\*(\*...)".
 
8. Выберите Сирепклиент. dll и Сшклиент. dll и нажмите кнопку "Добавить".
 
9. Выберите оба файла в обозреватель решений (они должны находиться в нижней части списка файлов) и измените значение параметра "Копировать в выходной каталог" в окно свойств в "Копировать всегда".
 
10. В верхней части файла добавьте следующий объект в существующий список операторов using: 
    ```
    using Microsoft.Tools.Deploy;
    using System.Net;
    ```
 
11. Внутри "статического void Main (...)" добавьте следующий код:
    ```
    RemoteDeployClient client = RemoteDeployClient.CreateRemoteDeployClient();
    client.Connect(new ConnectionOptions()
    {
        Credentials = new NetworkCredential("DevToolsUser", string.Empty),
        IPAddress = IPAddress.Parse(args[0])
    });
    client.RemoteDevice.DeleteFile(@"C:\Data\Users\DefaultAccount\AppData\Local\DevelopmentFiles\VSRemoteTools\x86\CoreCLR\mscorlib.ni.dll");
    ```
12. Сборка — > сборка решения
 
13. Откройте командную строку в папке, содержащей скомпилированный exe-файл (например, К:\мипрожектс\хололенсдеплойментфикс\бин\дебуг).
 
14. Запустите исполняемый файл и укажите IP-адрес устройства в качестве аргумента командной строки.  (Если подключение осуществляется через USB, можно использовать 127.0.0.1, в противном случае — IP-адрес устройства WiFi).  Например, "Хололенсдеплойментфикс 127.0.0.1"
 
15. После завершения работы средства без каких-либо сообщений (это займет всего несколько секунд), теперь можно выполнить развертывание и отладку Visual Studio 2017 или более поздней версии.  Продолжение использования этого средства не является обязательным.

Мы предложим дальнейшие обновления по мере их появления.

## <a name="issues-launching-the-microsoft-store-and-apps-on-hololens"></a>Проблемы при запуске Microsoft Store и приложений в HoloLens

>[!NOTE]
>Последнее обновление: 4/2 @ 10 — устранена проблема. 

При попытке запуска Microsoft Store и приложений в HoloLens могут возникнуть проблемы. Мы определили, что эта проблема возникает, когда фоновое обновление приложения развертывает более новую версию пакетов платформы в определенных последовательностях, в то время как одно или несколько зависимых приложений все еще работают. В этом случае автоматическое обновление приложения доставляет новую версию .NET Native Framework (версия 10.0.25531 в 10.0.27413) привела к тому, что приложения, работающие в, не будут правильно обновляться для всех работающих приложений, использующих более раннюю версию платформы.  Для обновления инфраструктуры используется следующий поток:-

1.  Новый пакет платформы загружается из магазина и устанавливается
2.  Все приложения, использующие более раннюю версию платформы, обновлены для использования более новой версии

Если шаг 2 был прерван до завершения, то приложения, для которых не была зарегистрирована более новая инфраструктура, не будут запущены из меню "Пуск".  Мы считаем, что в этом выпуске может повлиять любое приложение в HoloLens.

Некоторые пользователи сообщили, что закрытие зависших приложений и запуск других приложений, таких как центр отзывов, 3D Viewer или фотографии, решают проблему для них, однако это не работает 100% времени.

Корень привел к тому, что эта проблема не была вызвана обновлением, но ошибка в ОС, которая привела к неправильной обработке обновления .NET Native Framework. Мы рады сообщить, что мы обнаружили исправление и выпустили обновление (ОС версии 17763,380), содержащее исправление. 

Чтобы узнать, может ли устройство выполнить обновление, сделайте следующее:

1.  Перейдите к приложению "Параметры" и откройте "Обновление & безопасность".
2.  Щелкните "проверить наличие обновлений".
3.  Если доступно обновление до 17763,380, обновите эту сборку, чтобы получить исправление ошибки зависания приложения.
4.  После обновления до этой версии ОС приложения должны работать надлежащим образом.

Кроме того, по мере работы с каждым выпуском ОС HoloLens мы разместили образ ФФУ в центре загрузки Майкрософт по https://aka.ms/hololensdownload/10.0.17763.380 адресу. 

Если вы не хотите принимать обновление, мы выпустили новую версию Microsoft Store приложения UWP на 3/29. После получения обновленной версии хранилища:

1) Откройте магазин и убедитесь, что он загружается.
2) Чтобы открыть меню, используйте жест раскрытия.
3) Попытка открыть ранее работающие приложения
3) Если вы по-прежнему не можете запустить его, коснитесь и удерживайте значок неработающего приложения и выберите Удалить.
4) Ресинсталл эти приложения из магазина.

Если устройство по-прежнему не может загрузить приложения, вы можете загружать неопубликованные версию платформы .NET Native и среду выполнения через центр загрузки, выполнив следующие действия.

1)  Скачайте [этот ZIP-файл](http://download.microsoft.com/download/8/5/C/85C23745-794C-419D-B8D7-115FBCCD6DA7/netfx_1.7.zip) из центра загрузки Майкрософт.  При распаковке будут получены два файла.  Microsoft. NET. Native. Runtime. 1.7. appx и Microsoft. NET. Native. Framework. 1.7. appx
2)  Убедитесь, что устройство разблокировано в приложении dev.  Если вы еще не сделали этого, прежде чем приступать к [этим инструкциям](https://nam06.safelinks.protection.outlook.com/?url=https%3A%2F%2Fdocs.microsoft.com%2Fen-us%2Fwindows%2Fmixed-reality%2Fusing-the-windows-device-portal&data=02%7C01%7Cjalynch%40microsoft.com%7C3622a462ebd04870fccb08d6ae94cad6%7C72f988bf86f141af91ab2d7cd011db47%7C1%7C0%7C636888351416725140&sdata=ZB6Zdx9GV95PcU6FAVgWaP3eQNMsyIc%2FbNDEby3Sb8A%3D&reserved=0).
3)  Затем вы захотите перейти на портал устройств Windows.  Наша рекомендация заключается в том, чтобы сделать это через USB и сделать это, http://127.0.0.1:10080 введя в браузере.  
4)  Когда вы получите портал устройств Windows, вам нужно будет загрузить два скачанных файла.  Чтобы сделать это, необходимо перейти на левую боковую панель, пока не появится раздел "приложения" и щелкнуть "приложения".
5)  После этого появится экран, аналогичный приведенному ниже.  Вы хотите перейти к разделу «Install App» (Установка приложения) и перейти к расположению, в котором вы рассказали эти два APPX-файла.  Вы можете сделать это только по одному, поэтому после выбора первого из них щелкните "Go" в разделе "развертывание".  Затем сделайте это для второго APPX файла. 
  ![Портал устройств Windows для установки загруженного приложения](images/20190322-DevicePortal.png)<br>
6)  На этом этапе мы считаем, что ваши приложения должны снова начать работать, и вы также можете перейти в магазин.
7)  В некоторых случаях необходимо выполнить дополнительный шаг запуска приложения 3D Viewer, прежде чем затронутые приложения будут запущены. 

Мы ценим ваше терпение, так как мы пропустили этот процесс, чтобы устранить эту проблему, и мы надеемся продолжить работу с нашим сообществом для создания успешных смешанных возможностей.

## <a name="connecting-to-wifi"></a>Подключение к Wi-Fi

Во время & параметров OOBE имеется время ожидания учетных данных, равное 2 минутам. Имя пользователя и пароль необходимо указать в течение 2 минут, в противном случае поле "имя пользователя" будет автоматически сброшено.

Для ввода длинных паролей рекомендуется использовать клавиатуру Bluetooth.

>[!NOTE]
> Если во время OOBE выбрана неверная сеть, устройство потребуется полностью сбросить. Инструкции можно найти [здесь.](https://docs.microsoft.com/en-us/windows/mixed-reality/reset-or-recover-your-hololens#perform-a-full-device-recovery)

## <a name="visual-studio"></a>Visual Studio
* См. статью [Установка инструментов](install-the-tools.md) для наиболее актуальной версии Visual Studio, рекомендованной для разработки HoloLens.
* При развертывании приложения из Visual Studio в HoloLens может отобразиться следующее сообщение об ошибке: **Невозможно выполнить запрошенную операцию над файлом с открытым сопоставленным пользователем разделом. (Исключение из HRESULT: 0x800704C8)** . Если это произойдет, повторите попытку, и развертывание обычно завершится с ошибкой.

## <a name="emulator"></a>эмулятор
* Не все приложения в Microsoft Store совместимы с эмулятором. Например, Конкеры Титов и фрагменты не работают в эмуляторе.
* В эмуляторе нельзя использовать веб-камеру ПК.
* Функция интерактивной предварительной версии на портале устройств Windows не работает с эмулятором. Вы по-прежнему можете записывать видео и изображения смешанной реальности.

## <a name="unity"></a>Unity
* См. статью [Установка инструментов](install-the-tools.md) для наиболее актуальной версии Unity, рекомендуемой для разработки HoloLens.
* Известные проблемы с версией Unity HoloLens Technical Preview описаны на форумах [Hololens Unity](http://forum.unity3d.com/threads/known-issues.394627/).

## <a name="windows-device-portal"></a>Портал устройств Windows
* Функция интерактивной предварительной версии в записи смешанной реальности может демонстрировать несколько секунд задержки.
* На странице Виртуальный ввод элементы управления жестом и прокруткой в разделе Виртуальные жесты не работают. Их использование не будет действовать. Виртуальная клавиатура на той же странице работает правильно.
* После включения режима разработчика в параметрах может потребоваться несколько секунд, прежде чем включить портал устройств.

## <a name="api"></a>API
* Если приложение задает [точку фокусировки](focus-point-in-unity.md) за пользователем или нормальную камеру. Forward, голограммы не будут отображаться в фотографиях и видео в смешанной реальности. До тех пор пока эта ошибка не будет исправлена в Windows, если приложения активно устанавливают [фокусную точку](focus-point-in-unity.md) , она должна обеспечивать нормальную камеру (например, обычную =-Camera. Forward).

## <a name="xbox-wireless-controller"></a>Контроллер беспроводной связи Xbox
* Для использования с HoloLens необходимо обновить контроллер беспроводной сети Xbox. Прежде чем пытаться связать контроллер с HoloLens, убедитесь, что у вас есть [актуальная](https://support.xbox.com/xbox-one/accessories/update-controller-for-stereo-headset-adapter) версия.
* При перезагрузке HoloLens во время подключения контроллера беспроводной сети Xbox контроллер не будет автоматически подключаться к HoloLens. Кнопка "Обзор" будет мигать медленно, пока контроллер не завершит работу через 3 минуты. Чтобы немедленно подключиться к контроллеру, выключите контроллер, нажимая кнопку "переключить", пока не будет выключена лампочка. При повторном включении контроллера будет снова подключен к HoloLens.
* Если HoloLens попадает в ждущий режим во время подключения контроллера беспроводной сети Xbox, любые входные данные на контроллере будут пробудить HoloLens. Это можно предотвратить, выключив контроллер после завершения его использования.
