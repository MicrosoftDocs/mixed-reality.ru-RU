## <a name="experimenting-with-scale"></a><span data-ttu-id="4c5fb-132">Поэкспериментировав с масштабом</span><span class="sxs-lookup"><span data-stu-id="4c5fb-132">Experimenting with scale</span></span>

<span data-ttu-id="4c5fb-133">В некоторых случаях разработчики уже поэкспериментировали с изменение масштаба (путем изменения отображаемых «реальные» размер объекта), сохраняя единственную позицию объекта, для приближения ближе или рамками просмотра без фактическое перемещение объекта.</span><span class="sxs-lookup"><span data-stu-id="4c5fb-133">In some cases, designers have experimented with modifying the scale (by changing the displayed ‘real’ size of the object) while maintaining a single position of the object, to approximate an object getting closer or further to a viewer without any actual movement.</span></span> <span data-ttu-id="4c5fb-134">Это был протестирован в некоторых случаях, чтобы имитировать важнейшими Просмотр элементов, по-прежнему соблюдая потенциальные ограничения комфорта просмотра виртуального содержимого ближе, чем бы предложил «зоны комфорта».</span><span class="sxs-lookup"><span data-stu-id="4c5fb-134">This was tested in some cases as a way to simulate up-close viewing of items while still respecting potential comfort limitations of viewing virtual content closer than the “zone of comfort” would suggest.</span></span>

<span data-ttu-id="4c5fb-135">Это может создать несколько возможных артефактов в системе работы с тем не менее:</span><span class="sxs-lookup"><span data-stu-id="4c5fb-135">This can create a few possible artifacts in the experience however:</span></span>
* <span data-ttu-id="4c5fb-136">Виртуальные объекты, представляющие некий объект с размером «известные» средство просмотра, изменение масштаба без изменения положения приводит к конфликтующие визуальные подсказки — может по-прежнему глаза «см. в разделе» объект на некоторую глубину из-за подсказок vergence (см. в разделе [комфорта ](comfort.md) статье Подробнее об этом), но размер выступает в качестве monocular подсказки, объект может получение ближе.</span><span class="sxs-lookup"><span data-stu-id="4c5fb-136">For virtual objects that represent some object with a ‘known’ size to the viewer, changing the scale without changing the position leads to conflicting visual cues – the eyes may still ‘see’ the object at some depth due to vergence cues (see the [Comfort](comfort.md) article for more on this), but the size acts as a monocular cue that the object might be getting closer.</span></span> <span data-ttu-id="4c5fb-137">Эти конфликтующие подсказки привести к путать восприятие — средства просмотра часто просматривать объект как остаются на месте (из-за подсказки константы глубины), но быстро растет.</span><span class="sxs-lookup"><span data-stu-id="4c5fb-137">These conflicting cues lead to confused perceptions – viewers often see the object as staying in place (due to the constant depth cue) but growing rapidly.</span></span>
* <span data-ttu-id="4c5fb-138">В некоторых случаях изменение масштаба рассматривается как «надвигающихся» cue, вместо этого, объект может или не может появиться для изменения масштаба средство просмотра, когда могут быть напрямую сторону восприятия (которые могут быть неудобства ощущения).</span><span class="sxs-lookup"><span data-stu-id="4c5fb-138">In some cases, change of scale is seen as a ‘looming’ cue instead, where the object may or may not be seen to change scale by a viewer, but does appear to be moving directly toward the viewer’s eyes (which can be an uncomfortable sensation).</span></span>
* <span data-ttu-id="4c5fb-139">С помощью сравнения поверхностей в реальном мире, такие изменения масштабирования иногда рассматриваются как изменение положения по нескольким осям — объектов может показаться drop ниже вместо перемещения ближе (как в 2D проекции, трехмерных перемещения в некоторых случаях).</span><span class="sxs-lookup"><span data-stu-id="4c5fb-139">With comparison surfaces in the real world, such scaling changes are sometimes seen as changing position along multiple axes – objects may appear to drop lower instead of moving closer (similar in a 2D projection of 3D movement in some cases).</span></span>
* <span data-ttu-id="4c5fb-140">Наконец для объектов, не известных реальном мире размера (например произвольные формы с помощью произвольных размеров элементов пользовательского интерфейса, и т.д.), изменение масштаба может выступать в роли функционально возможность имитировать изменения в расстояние — средства просмотра не имеют столько ранее существовавших подсказки сверху вниз, на который понимать значение true, размер или расположение, объекта, чтобы масштаб могут обрабатываться как более важные ключевые.</span><span class="sxs-lookup"><span data-stu-id="4c5fb-140">Finally, for objects without a known ‘real world’ size (e.g. arbitrary shapes with arbitrary sizes, UI elements, etc.), changing scale may act functionally as a way to mimic changes in distance – viewers do not have as many preexisting top-down cues by which to understand the object’s true size or location, so the scale can be processed as a more important cue.</span></span>
