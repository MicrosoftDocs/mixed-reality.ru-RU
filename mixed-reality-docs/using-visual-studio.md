# <a name="using-visual-studio-to-deploy-and-debug"></a>С помощью Visual Studio для развертывания и отладки

Хотите ли вы использовать для разработки приложения смешанной реальности DirectX или Unity, Visual Studio будет использовать для отладки и развертывания. В этом разделе вы узнаете:
* Как развернуть приложения для вашей HoloLens или Windows Mixed Reality иммерсивных гарнитура с помощью Visual Studio.
* Сведения об использовании эмулятора HoloLens, встроенных в Visual Studio.
* Способы отладки приложений смешанной реальности.

## <a name="prerequisites"></a>Предварительные требования
1. См. в разделе [установить средства](install-the-tools.md) инструкции по установке.
2. Создайте новый проект приложения универсальной Windows в Visual Studio 2015 с обновлением 1 или Visual Studio 2017. C#, C++, и проекты JavaScript поддерживаются. (Или следуйте инструкциям, чтобы [Создание сборки приложения в Unity](holograms-100.md).)

## <a name="enabling-developer-mode"></a>Включение режима разработчика

Сначала включить **режим разработчика** на вашем устройстве, чтобы Visual Studio можно подключиться к нему.

### <a name="hololens"></a>HoloLens
1. Включите вашей HoloLens и передается на устройство.
2. Выполните жест [цветение](gestures.md#bloom), чтобы запустить главное меню.
3. При помощи **параметры** плитку и выполнять [жест касания](gestures.md#air-tap) жест. Выполните второе воздушное касание, чтобы разместить приложение в вашей среде. Приложение «Параметры» запустится после его размещения.
4. Выберите пункт меню **Обновить**.
5. Выберите пункт меню **Для разработчиков**.
6. Включите **Режим разработчика**. Это позволит вам [развертывание приложений из Visual Studio](using-visual-studio.md) для вашей HoloLens.
7. Дополнительно Прокрутите вниз, а также позволяют **портал устройств**. Это также позволит вам для подключения к [Windows Device Portal](using-the-windows-device-portal.md) на вашей HoloLens в веб-браузере.

### <a name="windows-pc"></a>Компьютер с Windows

Если вы работаете с гарнитуры смешанной реальности Windows, подключенном к ПК, необходимо включить **режим разработчика** на ПК.
1. Перейдите к **параметры**
2. Выберите **обновление и безопасность**
3. Выберите **для разработчиков**
4. Включить **режим разработчика**, читать заявление об отказе для выбранного параметра, а затем нажмите "Да", чтобы принять изменение.

## <a name="deploying-an-app-over-wi-fi---hololens-1st-gen"></a>Развертывание приложения по Wi-Fi - HoloLens (1-го поколения)
1. Выберите **x86** сборки конфигурации для приложения ![x86 создать конфигурацию в Visual Studio](images/x86setting.png)
2. Выберите **удаленный компьютер** в раскрывающемся меню целевого развертывания ![целевого объекта развертывания на удаленном компьютере в Visual Studio](images/remotemachinesetting.png)
3. Для C++ и проектов JavaScript, перейдите к **проект > Свойства > Свойства конфигурации > Отладка**. Для C# проектов, автоматически всплывающее будет диалоговое окно для настройки подключения.
  1. Введите IP-адрес устройства в **адрес** или **имя машины** поля. Найти IP-адрес на вашей HoloLens под **параметры > сеть и Интернет > Дополнительно**, или вы можете попросить Cortana «What is my IP-адреса?»
  2. Задайте режим проверки подлинности **универсальный (незашифрованный протокол)**![диалоговое окно удаленного подключения в Visual Studio](images/remotedeploy.png)
4. Выберите **Отладка > Начать отладку** для развертывания приложения и запуска отладки![Запуск без отладки в Visual Studio](images/deploynodebugging.png)
5. При первом развертывании приложения для вашей HoloLens с ПК, вам будет предложено ввести ПИН-код. Выполните **связывание устройства** приведенным ниже инструкциям.

Если изменяется HoloLens IP-адрес, IP-адрес конечного компьютера можно изменить, выбрав **проект > Свойства > Свойства конфигурации > Отладка**

## <a name="deploying-an-app-over-usb---hololens-1st-gen"></a>Развертывание приложения по USB - HoloLens (1-го поколения)
1. Выберите **x86** сборки конфигурации для приложения ![x86 создать конфигурацию в Visual Studio](images/x86setting.png)
2. Выберите **устройства** в раскрывающемся меню целевого развертывания![развертывания устройства в Visual Studio](images/buildsettingsusbdeploy.png)
3. Выберите **Отладка > Начать отладку** для развертывания приложения и запуска отладки![Запуск без отладки в Visual Studio](images/deploynodebugging.png)
4. При первом развертывании приложения для вашей HoloLens с ПК, вам будет предложено ввести ПИН-код. Выполните **связывание устройства** приведенным ниже инструкциям.

## <a name="deploying-an-app-to-your-local-pc---immersive-headset"></a>Развертывание приложения на локальный компьютер - гарнитура иммерсивных

Следуйте этим инструкциям при использовании Windows Mixed Reality иммерсивных гарнитура, который подключается к Компьютеру или [симулятор смешанной реальности](using-the-windows-mixed-reality-simulator.md). В этом случае развертывание и запуск приложения на локальном Компьютере.
1. Выберите **x86** или **x64** сборки конфигурации для приложения
2. Выберите **локальный компьютер** в раскрывающемся меню целевого развертывания
3. Выберите **Отладка > Начать отладку** для развертывания приложения и запуска отладки

## <a name="pairing-your-device---hololens-1st-gen"></a>Связывание устройства - HoloLens (1-го поколения)

При первом развертывании приложения из Visual Studio для вашей HoloLens, вам будет предложено ввести ПИН-код. HoloLens, создавать ПИН-кода, запустив приложение параметры, перейдите к **обновления > для разработчиков** и коснуться **пары**. ПИН-код будет отображаться на вашей HoloLens; Введите этот ПИН-код в Visual Studio. По завершении связывание коснитесь **сделать** на вашей HoloLens, чтобы закрыть диалоговое окно. Этот компьютер теперь сопряжен с HoloLens и можно будет автоматически развертывать приложения. Повторите эти шаги для каждого последующих ПК, который используется для развертывания приложений в вашей HoloLens.

К паре отмены запуска вашего HoloLens со всех компьютеров, он был связан, **параметры** приложение, перейдите к **обновления > для разработчиков** и коснуться **Очистить**.

4. Выберите **Отладка > Начать отладку** для развертывания приложения и запуска отладки![Запуск без отладки в Visual Studio](images/deploynodebugging.png)

## <a name="graphics-debugger"></a>Отладчик графики

Средства диагностики графики Visual Studio очень полезны в тех случаях, при написании и оптимизация Holographic приложения. См. в разделе [диагностики графики Visual Studio на сайте MSDN](https://msdn.microsoft.com/library/hh315751.aspx) полные сведения.

**Чтобы запустить отладчик графики**
1. Следуйте приведенным выше инструкциям для устройства или эмулятора
2. Перейдите к **Отладка > графики > Запуск диагностики**
3. В первый раз это сделать с помощью HoloLens, может появиться ошибка «доступ запрещен». Перезагрузите вашей HoloLens, чтобы обновленные разрешения вступают в силу, и повторите попытку.

## <a name="profiling"></a>Профилирование

Средства профилирования Visual Studio позволяют анализировать использование приложения производительности и ресурсов. Сюда входят средства для оптимизации ЦП, памяти, графики и использования сети. См. в разделе [запуск средств диагностики без отладки на сайте MSDN](https://msdn.microsoft.com/library/dn957936.aspx) полные сведения.

**Для запуска средства профилирования с HoloLens**
1. Следуйте приведенным выше инструкциям для устройства или эмулятора
2. Перейдите к **Отладка > запуск средств диагностики без отладки...**
3. Выберите Инструменты, которые вы хотите использовать
4. Нажмите кнопку **запуск**
5. В первый раз это сделать с помощью HoloLens, может появиться ошибка «доступ запрещен». Перезагрузите вашей HoloLens, чтобы обновленные разрешения вступают в силу, и повторите попытку.

## <a name="debugging-an-installed-or-running-app"></a>Отладка приложения установлен или не запущен

Visual Studio можно использовать для отладки приложение универсальной Windows, которое установлено без развертывания из проекта Visual Studio. Это полезно в том случае, если требуется отладка установленного пакета приложения, или если вы хотите выполнить отладку приложения, на котором уже выполняется.
1. Перейдите к **Отладка -> другие отладки целевые объекты "->" отлаживать установленный пакет приложения**
2. Выберите **удаленный компьютер** целевой объект для HoloLens или **локального компьютера** для иммерсивную.
3. Введите устройства **IP-адрес**
4. Выберите **универсальной** режим проверки подлинности
5. В окне отображаются запущен и неактивные приложения. Выберите тот, который вы хотите отладки.
6. Выберите тип кода для отладки (управляемый, машинный код, Mixed)
7. Нажмите кнопку **присоединить** или **запуск**
