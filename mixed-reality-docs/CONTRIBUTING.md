# <a name="contributing-to-windows-mixed-reality-developer-documentation"></a>Дополнение к документации разработчика Windows Mixed Reality

Добро пожаловать в [общедоступный репозиторий для документации разработчика Windows Mixed Reality](https://github.com/MicrosoftDocs/mixed-reality/tree/master/mixed-reality-docs)! Все статьи, можно создавать и изменять в этом репозитории **будет доступно для просмотра.** 

Документы по Windows Mixed Reality теперь представлены на платформе docs.microsoft.com, которая использует GitHub flavored Markdown (с помощью функций Markdig). По сути, содержимое, редактировать в этот репозиторий возвращает преобразуются в отформатирован и стилизованные страниц, которые отображаются в https://docs.microsoft.com/windows/mixed-reality. 

Эта страница описывает основные шаги и правила участия, а также ссылки на основные сведения о Markdown. Спасибо за Ваш вклад!

## <a name="before-you-start"></a>Прежде чем начать

Если вы еще не, вам необходимо будет [создайте учетную запись GitHub](https://github.com/join).

>[!NOTE]
>Если вы являетесь сотрудником корпорации Майкрософт, свяжите учетную запись GitHub, чтобы ваш псевдоним Майкрософт на [портал с открытым исходным кодом Microsoft](https://repos.opensource.microsoft.com/). Присоединяйтесь к **«Microsoft»** и **«MicrosoftDocs»** организаций).

При настройке учетной записи GitHub, мы также рекомендуем эти меры безопасности:
- Создание [надежный пароль для учетной записи Github](https://github.com/settings/admin).
- Включить [двухфакторной проверки подлинности](https://github.com/settings/two_factor_authentication/configure).
- Сохранить ваши [коды восстановления](https://github.com/settings/auth/recovery-codes) в надежном месте.
- Обновление вашей [параметры общего профиля](https://github.com/settings/profile).
   - Задайте имя и попробуйте задать вашей *общедоступный почтовый* для *больше не показывать Мой адрес электронной почты*.
   - Мы рекомендуем вам добавить фотографию профиля как эскиза будет отображаться на страницах проекта docs, к которым вы внести свой вклад.
- Если вы планируете использовать рабочий процесс командной строки, рекомендуется настроить [Git Credential Manager для Windows](https://github.com/Microsoft/Git-Credential-Manager-for-Windows/releases/latest) , не нужно вводить пароль каждый раз внести свой вклад.

Эти действия важно, как публикации системы привязывается к GitHub и отобразится как автор или участник для каждой статьи, используя свой псевдоним GitHub.

## <a name="editing-an-existing-article"></a>Изменение существующей статьи

Используйте следующий рабочий процесс для обновления *существующей статьи* на сайте GitHub в веб-браузере:

1. Перейдите к статье, которую вы хотите изменить в папке «смешанный реальности документы».
2. Нажмите кнопку редактирования (значок карандаша) в правом верхнем углу. Это автоматически будет ответвлять высвобождаемые ветви «master» ветви.

   ![Изменение статьи.](images/editpage.png)
3. Изменить содержимое статьи (см. в разделе [«Основные сведения о Markdown»](#markdown-basics) ниже рекомендации).
4. Обновите метаданные в виде соответствующих в верхней части каждой статьи:
   * Заголовок: Это заголовок страницы, который отображается на вкладке браузера при просмотре статьи. Так как это используется для оптимизации поисковой системы и индексирования, не следует изменять заголовок, если не требуется (хотя это не столь важно, перед отправкой общедоступной документации).
   * description: Записать краткое описание содержимое статьи. Это также помогает при оптимизации поисковой системы и обнаружения.
   * Автор: Если вы являетесь владельцем основной части страницы, добавьте здесь свой псевдоним GitHub.
   * ms.author: Если вы являетесь владельцем основной части страницы, добавьте Microsoft здесь псевдоним (не требуется @microsoft.com, просто псевдоним).
   * ms.date: Обновляет дату, при добавлении основных содержимого на страницу, но не для исправления, например пояснение, форматирование, Грамматика, или проверки орфографии.
   * ключевые слова: Ключевые слова помочь в оптимизации поисковой системы (оптимизация для поисковых систем). Добавьте ключевые слова, разделяя их точкой с запятой и пробелом, относящихся к вашу статью (но знаки препинания, после ключевого слова последнего в списке); не нужно добавить глобальные ключевые слова, которые применяются ко всем статьям, как осуществляется в другом месте. 
5. После завершения изменения статьи, прокрутите вниз и выберите **предложить изменение файла** кнопки.
6. На следующей странице щелкните **создать запрос на Вытягивание** выполнить слияние ветви автоматически создается в «master».
7. Повторите предыдущие шаги для следующей статьи, которые вы хотите изменить.

## <a name="creating-a-new-article"></a>Создание новой статьи

Используйте следующий рабочий процесс для *создание новых статей* в репозитории документации на сайте GitHub в веб-браузере:

1. Создайте вилку ветвь, «master» MicrosoftDocs/смешанной реальности (с помощью **вилки** кнопки в правом верхнем углу).

   ![Вилка главной ветви.](images/forkbranch.png)
2. В папке «смешанный реальности документы», щелкните **создать новый файл** кнопки в правом верхнем углу.
3. Создайте имя страницы для статьи (использовать дефисами вместо пробелов и не используйте знаки препинания или апострофы) и добавьте «.md»

   ![Присвойте имя новой страницы.](images/newpagetitle.PNG)
   
   >[!IMPORTANT]
   >Убедитесь, что вы создаете новую статью из в папке «смешанный, реальность документы». Это можно проверить путем проверки «/ смешанного реальности docs /» в строке имя нового файла.

4. В верхней части новой страницы добавьте следующий блок метаданных:

   ```md
   ---
   title:
   description:
   author:
   ms.author:
   ms.date:
   ms.topic: article
   keywords:
   ---
   ```

5. Заполните поля соответствующими метаданными, следуя инструкциям в [выше разделе](#editing-an-existing-article).
6. Записи статьи содержимого с помощью [основные сведения о Markdown](#markdown-basics).
7. Добавить `## See also` раздел находится в нижней части статьи со ссылками на другие соответствующие статьи.
8. Закончив, нажмите кнопку **фиксации нового файла**.
9. Нажмите кнопку **новый запрос на Вытягивание** и слияние ветви «master» вилки в MicrosoftDocs/смешанной реальности «master» (Убедитесь, что стрелка указывает правильный способ использования).

   ![Создание запроса на Вытягивание из вилки в MicrosoftDocs/смешанной реальности](images/pr_to_master.PNG)

### <a name="adding-tables"></a>Добавление таблиц

Из-за как таблицах стилей на сайте docs.microsoft.com они не имеют границы или пользовательские стили, даже если вы попытаетесь встроенных таблиц CSS. Он будет работать на короткое время, но в конечном итоге платформы будут удалены из таблицы стилей. Поэтому Подготовьтесь заранее и усложнять таблиц. [Здесь — это сайт, упрощает таблиц Markdown](http://www.tablesgenerator.com/markdown_tables).

[Расширении Docs Markdown для Visual Studio Code](https://docs.microsoft.com/teamblog/docs-extension) также делает таблица создания простой, если вы используете [Visual Studio Code (см. ниже)](#using-visual-studio-code) внесение изменений в документации.

### <a name="adding-images"></a>Добавление изображений

Вам потребуется передать свои образы в смешанном реальности документы или папки «images» в репозитории, и затем ссылаться на них соответствующим образом в этой статье. Образы будут автоматически отображаться в полном размере, означающее, если изображение имеет большой размер, будет заполнено всю ширину элемента статьи. Таким образом рекомендуется предварительно изменения размера изображения перед их отправкой. Рекомендуемую ширину — в диапазоне от 600 до 700 точек, то, что следует размер вверх или вниз, если это плотных снимок экрана или небольшую часть снимка экрана, соответственно.

>[!IMPORTANT]
>Образы можно передать только в разветвление репозитория перед слиянием. Таким образом, если планируется добавление изображений в статью, вам необходимо будет [использование Visual Studio Code](#using-visual-studio-code) сначала Добавление изображений в папку «изображения» в вилку или убедитесь, что вы выполнили следующие действия в веб-браузере:
>
>1. Создании разветвления в репозиторий MicrosoftDocs/смешанной реальности.
>2. Редактировать статьи в вилке.
>3. Переданных изображений, который вы ссылаетесь в своей статье в папку «смешанный реальность документы и изображения» в вилке.
>4. Создан **запрос на Вытягивание** для слияния вилке ветвью «master» MicrosoftDocs/смешанной реальности.
>
>Чтобы узнать, как настроить разветвления репозитория, следуйте инструкциям для [создания новой статьи](#creating-a-new-article).

## <a name="previewing-your-work"></a>Предварительный просмотр рабочей

Во время редактирования в GitHub с помощью веб-браузер, можно щелкнуть **предварительной версии** вкладки в верхней части страницы, чтобы просмотреть данные перед фиксацией. 

>[!NOTE]
>Предварительный просмотр изменений на review.docs.microsoft.com доступен только для сотрудников корпорации Майкрософт

Сотрудники корпорации Майкрософт: как только Ваш вклад были объединены в ветви «master», можно увидеть, как будет выглядеть документации перед отправкой его в открытый https://review.docs.microsoft.com/windows/mixed-reality?branch=master (найти статью с помощью оглавление в левом столбце).

## <a name="editing-in-the-browser-vs-editing-with-a-desktop-client"></a>Редактирование в браузере и редактирование с помощью настольного клиента

Редактирование в браузере является самым простым способом для быстрого изменения, однако есть несколько недостатков:

- Вы не получаете проверки орфографии.
- Полученный любой смарт связывание с другими статьями (необходимо вручную ввести имя файла из статьи).
- Это может быть хлопот для отправки и ссылок на изображения.

Если вы бы не иметь появляющихся подобных проблем, вам может потребоваться рабочего стола клиента, например [Visual Studio Code](https://code.visualstudio.com/) пару [полезные расширения](#useful-extensions) для участия в разработке документации.

## <a name="using-visual-studio-code"></a>С помощью Visual Studio Code

По причинам, в списке [выше](#editing-in-the-browser-vs-editing-with-a-desktop-client), вы можете предпочесть, изменяя с помощью настольного клиента документации вместо веб-браузер. Мы рекомендуем использовать [Visual Studio Code](https://code.visualstudio.com/).

### <a name="setup"></a>Установка

Выполните следующие действия, чтобы настроить Visual Studio Code для работы с этим репозиторием.

1. В веб-браузере:
    1. Установка [Git для ПК](https://git-scm.com/downloads).
    2. Установка [Visual Studio Code](https://code.visualstudio.com/).
    3. [Разветвление MicrosoftDocs/смешанной реальности](#creating-a-new-article) Если у вас еще нет.
    4. В этой вилке щелкните **клонировать или скачать** и скопируйте URL-адрес.
2. Создайте локальный клон разветвления в Visual Studio Code:
    1. Из **представление** меню, выберите **палитру команд**.
    2. Тип «Git:Clone.»
    3. Вставьте скопированный URL-адрес.
    4. Укажите место для сохранения копии на вашем Компьютере.
    5. Нажмите кнопку **откройте репозиторий** во всплывающем окне.

### <a name="editing-documentation"></a>Редактирование документации

Используйте следующий рабочий процесс для внесения изменений в документации с помощью Visual Studio Code:

>[!NOTE]
>Все рекомендации для [редактирование](#editing-an-existing-article) и [создание](#creating-a-new-article) статей и [основы редактирования разметки Markdown](#markdown-basics), выше применяется при использовании Visual Studio Code, а также.

1. Убедитесь, что клонированный вилки в официальный репозиторий.
   1. В веб-браузер, создайте запрос на Вытягивание, чтобы синхронизировать последние изменения других участников в MicrosoftDocs/смешанной реальности «master», в свою вилку (Убедитесь, что стрелка указывает правильный способ).
      
      ![Синхронизировать изменения из MicrosoftDocs/смешанной реальности вилке](images/sync_repos.PNG)
   2. В Visual Studio Code нажмите кнопку синхронизации для синхронизации только что обновленный вилки к локальному клону.
      
      ![Нажмите кнопку "Синхронизация"](images/sync_clone.png)
2. Создание или изменение статьи в клонированном репозитории с помощью Visual Studio Code.
   1. Измените одну или несколько статей (Добавить папку «изображения» образов, при необходимости).
   2. **Сохранить** изменения в **Explorer**.
      
      ![Выберите «Сохранить все» в обозревателе](images/explorer_save.png)
   3. **Фиксация всех изменений** изменения в **системы управления версиями** (записи сообщения фиксации при появлении запроса).
      
      ![Выберите «Сохранить все» в системе управления версиями](images/source_control_commit.png)
   4. Нажмите кнопку **синхронизации** кнопку, чтобы синхронизация внесенных изменений обратно в исходный репозиторий (разветвление на GitHub).
      
      ![Нажмите кнопку "Синхронизация"](images/sync_back.png)
3. В веб-браузер, создайте запрос на Вытягивание, чтобы синхронизировать изменения в вилке к MicrosoftDocs/смешанной реальности «master» (Убедитесь, что стрелка указывает правильный способ использования).

   ![Создание запроса на Вытягивание из вилки в MicrosoftDocs/смешанной реальности](images/pr_to_master.PNG)

### <a name="useful-extensions"></a>Полезные расширения

Следующие расширения Visual Studio Code очень полезны при редактировании документации:

- [Расширения docs Markdown для Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=docsmsft.docs-authoring-pack) -использование **Alt + M** откроется меню создания параметрами, такими как документации:
   - Справочник по и поиска образов, отправленные вами.
   - Добавление форматирования, такие как списки, таблицы, и вызовами документации Майкрософт, например `>[!NOTE]`.
   - Поиск и ссылки на внутренние ссылки и закладки (ссылки на разделы на странице).
   - Ошибки форматирования будут выделены (наведите указатель мыши ошибки Дополнительные сведения).
- [Код проверки орфографии](https://marketplace.visualstudio.com/items?itemName=streetsidesoftware.code-spell-checker) -слов с ошибками будут подчеркнуты; щелкните правой кнопкой мыши на неправильно написанное слово можно изменить или сохранить его в словарь.
