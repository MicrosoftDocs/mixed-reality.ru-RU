## <a name="holographic-rendering"></a>Holographic отрисовки

Ключом к отрисовке holographic является понимание того ли вы к просмотру в предыдущую изображение, показанное HoloLens, позволяющее пользователям см. в разделе физического мира и вашей голограммы вместе--или непрозрачное изображение, показанное иммерсивных гарнитуры смешанной реальности Windows, блокирующий всему миру.

Устройства с **сквозная отображает**, например [HoloLens](hololens-hardware-details.md), добавить свет во всем мире. Области черных пикселей являются полностью прозрачными, а более яркие пикселы, все чаще непрозрачны. Поскольку света с дисплеев добавляется свет из реального мира, белые пиксели являются немного прозрачным.

Stereoscopic отрисовки предоставляет один ключевой глубины для вашей голограммы, добавление [заземление эффекты](interaction-fundamentals.md) может помочь пользователям см. в статье, проще какие поверхности голограмма приближается. Один из способов заземление — для добавления в области рядом свечение вокруг голограмма и визуализировать тень от этого свечения. Таким образом ваш тени отображается для вычитания light из среды. [Пространственные звук](spatial-sound.md) другого индикатора крайне важно глубины, позволяя пользователям выводы о расстояние и относительное расположение голограмма.

Устройства с **непрозрачный отображает**, например [Windows Mixed Reality иммерсивную](immersive-headset-hardware-details.md), заблокировать всему миру. Области черных пикселей являются сплошным черным цветом, а любой другой цвет представляются как цвета для пользователя. Приложение отвечает за отрисовку, которое все, что пользователь видит. Это позволяет еще более важно поддерживать постоянное частоты, чтобы пользователи могли комфортную работу.

## <a name="predicted-rendering-parameters"></a>Прогнозируемые параметров подготовки отчетов

Смешанный гарнитуры реальности (HoloLens и иммерсивную) постоянно отслеживать положение и ориентацию головы пользователя по отношению к окружающей среды. Как приложение начинает подготавливать его следующего кадра, система прогнозирует, где head пользователя будет в будущем в тот момент, когда отображался кадр на вывод разделов. На основании прогноза, система вычисляет преобразование представления и проекции, которое требуется использовать для этого кадра. Приложение **необходимо использовать эти преобразования для получения правильных результатов**; Если системное преобразования не используются, полученный в результате образ не полностью соответствуют реального мира, что приводит к discomfort пользователя.

Обратите внимание на то, что для точного прогнозирования, когда новый кадр достигнет дисплеев, система постоянно измерение действующие задержки end-to-end конвейера отрисовки приложения. Хотя система изменяется так, чтобы длина конвейера отрисовки, может повысить стабильность голограмма, оставив этого конвейера, как можно более короткими.

Приложения, использующие дополнительные методы для расширения системы прогноза можно переопределить преобразования представления и проекции системы. Эти приложения необходимо необходимо по-прежнему использовать системное преобразования в качестве основы для своих пользовательских преобразований для выдают значащие результаты.

## <a name="other-rendering-parameters"></a>Другие параметры подготовки к просмотру

При отрисовке определенного кадра, система указывает окна просмотра задний буфер, в которой следует нарисовать приложения. Окно просмотра часто меньше, чем полный размер буфера кадров. Независимо от размера окна просмотра после отрисовки кадра приложением, система upscales изображение, чтобы заполнить весь дисплеев.

Для приложений, которые оказываются невозможно воспроизвести частотой необходимые обновления [можно настроить параметры системы подготовки к просмотру](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfiguration#Windows_Graphics_Holographic_HolographicViewConfiguration) для сокращения нехватки памяти и подготовки к просмотру затрат за счет присвоения псевдонимов Повышенная пикселей. Форматом буфера фона также можно изменить, что для некоторых приложений может помочь повысить пропускную способность памяти пропускной способности и пикселей.

Пирамиды обзора подготовки отчетов, разрешение и частоте кадров, в котором приложения запрашивается для отрисовки, также может измениться из фрейма и могут отличаться между левым и правым глаза. Например, если [смешанный реальности записи](mixed-reality-capture.md) (MRC) активен и [фото и видео камеры Просмотр конфигурации](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfigurationKind#Windows_Graphics_Holographic_HolographicViewConfigurationKind) не согласились с заходом, один глаз могут отображаться с FOV большего размера или разрешения.

Для любого заданного кадра, приложение *необходимо* отрисовки, используя представление преобразования, преобразование проекции и разрешение окна просмотра, предоставляемые системой. Кроме того приложение должно нельзя предполагать, что любой параметр подготовки к просмотру или представление остается фиксированным из одного фрейма. Ядра, такие как Unity обрабатывать все эти преобразования для вас в свои собственные объекты камеры, чтобы всегда соблюдать физическое перемещение пользователей и состояние системы. Если приложение допускает виртуальный перемещения пользователя по всему миру (например, используя джойстик на игровой), можно добавить родительского объекта тестовой платформы выше камеры, который она перемещается. Это приводит к камере, чтобы отразить как виртуального и физического перемещения пользователя. Если приложение изменяет представление преобразования, преобразование проекции или измерения окна просмотра, предоставляемые системой, он должен сообщить системе, вызвав соответствующий [переопределить API](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicCameraPose#Windows_Graphics_Holographic_HolographicCameraPose).

В целях повышения стабильности holographic отрисовку приложения должен предоставить Windows каждого кадра буфер глубины, она используется для подготовки к просмотру. Если приложение предоставляет буфер глубины, он должен иметь значения согласовано глубины, с глубиной, указывается в метрах от камеры. Это позволяет системе использовать ваши данные на точку глубины лучше стабилизация содержимое head пользователя заканчивается немного смещения из прогнозируемых расположения. Если вы не может предоставить вашей буфер глубины, можно указать точку фокуса и норм, определение плоскость, которые снижают через большую часть содержимого. Если буфер глубины и плоскость фокус, система может использовать оба. В частности полезно предоставить буфер глубины и точку фокуса, который включает в себя вектора скорости, когда приложение отображает голограммы движущихся.

## <a name="holographic-cameras"></a>Holographic камеры

Windows Mixed Reality введено понятие **holographic камеры**. Holographic камеры похожи на традиционные камеры в 3D-графики тексты: они определяют обоих внешних (положение и ориентацию) и свойства внутренние камеры. (Например:, поля вида используется для просмотра виртуального трехмерной сцены.) В отличие от традиционных трехмерной камеры приложение не управления позиции, ориентации и внутренние свойства камеры. Вместо этого положение и ориентацию holographic камеры неявно управляет перемещения пользователя. Перемещение пользователя передается в приложение на основе фрейм за фреймом через преобразование представления. Аналогично внутренние свойства камеры определяются калиброванных оптика устройства и ретрансляционную фрейма через преобразование проекции.

В целом приложения будет отображаться для одного стерео камеры. Тем не менее надежные отрисовки цикл будет поддерживать несколько камеры и будет поддерживать mono и стерео камеры. Например, система может запрос приложения для подготовки к просмотру с точки зрения альтернативный, когда пользователь активирует компонент как [смешанный захвата реальности](mixed-reality-capture.md) (MRC), зависит от формы рассматриваемый гарнитура. Приложения, которые может поддерживать несколько камеры получить их, [подтвердить](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfiguration#Windows_Graphics_Holographic_HolographicViewConfiguration) для [вид](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfigurationKind#Windows_Graphics_Holographic_HolographicViewConfigurationKind) камер, они могут поддерживать.
