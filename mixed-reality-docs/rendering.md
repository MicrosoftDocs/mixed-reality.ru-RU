## <a name="holographic-rendering"></a>Holographic отрисовки

Ключом к отрисовке holographic является понимание того ли вы к просмотру в предыдущую изображение, показанное HoloLens, который позволяет пользователю видеть физическом мире и вашей голограммы вместе - или непрозрачный отображения, например гарнитура иммерсивных Windows Mixed Reality, какие блоки мира.

Устройства с **сквозная отображает**, например [HoloLens](hololens-hardware-details.md), добавить свет во всем мире. Области черных пикселей будет полностью прозрачным, пока ярче пикселей не более прозрачен. Поскольку света с дисплеев добавляется свет из реального мира, даже белые пиксели являются немного прозрачным.

Stereoscopic отрисовки предоставляет один ключевой глубины для вашей голограммы, добавление [заземление эффекты](interaction-fundamentals.md) может помочь пользователям см. в статье, проще какие поверхности голограмма приближается. Один из способов заземление — Добавление свечение вокруг голограмма в области рядом и последующей визуализации тень от этого свечения. Таким образом ваш отбрасывается тень для вычитания light из среды. [Пространственные звук](spatial-sound.md) может быть другой ключевой крайне важно глубины, позволяя пользователям выводы о расстояние и относительное расположение голограмма.

Устройства с **непрозрачный отображает**, например [Windows Mixed Reality иммерсивную](immersive-headset-hardware-details.md), заблокировать всему миру. Области черных пикселей будет сплошным черным цветом, и любой другой цвет будет отображаться как цвета для пользователя. Приложение отвечает за визуализацию все, что пользователь будет видеть, поэтому еще более важно поддерживать постоянное частоты, чтобы пользователи могли комфортную работу.

## <a name="predicted-rendering-parameters"></a>Прогнозируемые параметров подготовки отчетов

Смешанный гарнитуры реальности (HoloLens и иммерсивную) постоянно отслеживать положение и ориентацию головы пользователя по отношению к окружающей среды. Как приложение начинает подготавливать его следующего кадра, система прогнозирует, где head пользователя будет в будущем в тот момент, когда, будут отображаться на дисплеях кадр. На основании прогноза, система вычисляет преобразование представления и проекции, которое требуется использовать для этого кадра. Приложение **необходимо использовать эти преобразования для получения правильных результатов**; Если системное преобразования не используются, полученный в результате образ не полностью соответствуют реального мира, что приводит к discomfort пользователя.

Обратите внимание на то, что для точного прогнозирования, когда новый кадр достигнет дисплеев, система постоянно измерение действующие задержки end-to-end конвейера отрисовки приложения. Хотя система скорректирует длине конвейера отрисовки, можно и дальше улучшать голограмма стабильность, оставив этого конвейера, как можно более короткими.

Приложения, использующие дополнительные методы для расширения системы прогноза можно переопределить преобразования представления и проекции системы. Эти приложения необходимо необходимо по-прежнему использовать системное преобразования в качестве основы для своих пользовательских преобразований для выдают значащие результаты.

## <a name="other-rendering-parameters"></a>Другие параметры подготовки к просмотру

При отрисовке определенного кадра, система указывает окна просмотра задний буфер, в которой следует нарисовать приложения. Окно просмотра часто будет меньше, чем полный размер буфера кадров. Независимо от размера окна просмотра когда кадр визуализирован приложением объект, система увеличить изображение, чтобы заполнить весь отображается.

Для приложений, которые оказываются невозможно воспроизвести частотой необходимые обновления [можно настроить параметры системы подготовки к просмотру](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfiguration#Windows_Graphics_Holographic_HolographicViewConfiguration) для уменьшения нехватки памяти и/или отрисовки затрат за счет присвоения псевдонимов Повышенная пикселей. Форматом буфера фона также можно изменить, что для некоторых приложений может помочь повысить пропускную способность памяти пропускной способности и пикселей.

Пирамиды обзора подготовки отчетов, разрешение и частоте кадров, в котором приложения запрашивается для отрисовки также может измениться из фрейма и могут отличаться между левым и правым глаза. Например, если [смешанный реальности записи](mixed-reality-capture.md) (MRC) активен и [фото и видео камеры Просмотр конфигурации](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfigurationKind#Windows_Graphics_Holographic_HolographicViewConfigurationKind) не согласились с заходом, один глаз может быть выведено с большего размера FOV или разрешения.

Для любого заданного кадра, приложение *необходимо* отрисовки, используя представление преобразования, преобразование проекции и разрешение окна просмотра, предоставляемые системой. Кроме того приложение должно нельзя предполагать, что любой параметр подготовки к просмотру или представления остается неизменным из одного фрейма. Ядра, такие как Unity обрабатывать все эти преобразования для вас в свои собственные объекты камеры, чтобы всегда соблюдать физическое перемещение пользователей и состояние системы. Если приложение допускает виртуальный перемещения пользователя по всему миру (например, используя джойстик на игровой), можно добавить родительского объекта «оборудование» выше камера, которая перемещается, приводит к камере, чтобы отразить оба пользователя виртуального и физического перемещения. Если приложение изменяет представление преобразования, преобразование проекции или измерения окна просмотра, предоставляемые системой, он должен сообщить системе, вызвав соответствующий [переопределить API](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicCameraPose#Windows_Graphics_Holographic_HolographicCameraPose).

В целях повышения стабильности holographic отрисовку приложения должен предоставить Windows каждого кадра буфер глубины, она используется для подготовки к просмотру. Если приложение предоставляет буфер глубины, он должен иметь значения согласовано глубины, с глубиной, указывается в метрах от камеры. Это позволяет системе использовать ваши данные на точку глубины лучше стабилизация содержимое head пользователя заканчивается немного смещения из прогнозируемых расположения. Если вы не может предоставить вашей буфер глубины, следует вместо этого предоставить точку фокуса и норм, определение плоскость, которые снижают через большую часть содержимого. Если буфер глубины и плоскость фокус, система может использовать оба. В частности можно предоставить буфер глубины и точку фокуса, который включает в себя вектора скорости, когда приложение отображает голограммы движущихся.

## <a name="holographic-cameras"></a>Holographic камеры

Windows Mixed Reality введено понятие **holographic камеры**. Holographic камеры похожи на традиционные камеры в 3D-графики тексты: они определяют обоих внешних (положение и ориентацию) и свойства внутренние камеры (ex: поле зрения) используется для просмотра виртуального трехмерной сцены. В отличие от традиционных трехмерной камеры приложение не управления позиции, ориентации и внутренние свойства камеры. Вместо этого положение и ориентацию holographic камеры неявно управляет перемещения пользователя. Перемещение пользователя передается в приложение на основе фрейм за фреймом через преобразование представления. Аналогично внутренние свойства камеры определяются калиброванных оптика устройства и ретрансляционную фрейма через преобразование проекции.

В общем приложении будет отрисовки для одного стерео камеры. Тем не менее цикл надежных подготовки отчетов должен поддерживать несколько камеры и должен поддерживать mono и стерео камеры. Например, система может задать приложение для подготовки к просмотру с точки зрения альтернативный, когда пользователь активирует компонент как [смешанный реальности записи](mixed-reality-capture.md) (MRC), зависит от формы гарнитура в вопросе. Приложения, которые может поддерживать несколько камеры получить их, [подтвердить](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfiguration#Windows_Graphics_Holographic_HolographicViewConfiguration) для [вид](https://docs.microsoft.com/uwp/api/Windows.Graphics.Holographic.HolographicViewConfigurationKind#Windows_Graphics_Holographic_HolographicViewConfigurationKind) камер, они могут поддерживать.
